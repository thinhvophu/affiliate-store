# Feature Specification: About Page
**ID:** `F0010`
**Epic/Module:** Compliance & Trust
**Status:** Approved

---

## 1. Business Objective
Deliver a static trust-signal page at `/ve-chung-toi/` that explains the site's mission,
who runs it, and how products are chosen, restates the affiliate disclosure, and exposes a
contact point. Beyond the trust value, this feature closes a live gap: the header navigation
link "Về chúng tôi" and the footer "Giới thiệu" link both point at `/ve-chung-toi/`, which
currently has no route and returns a 404. Shipping this page removes that broken link and
strengthens the site's E-E-A-T signals for Google's quality guidelines on affiliate sites.

## 2. Scope
### In Scope
* Static (SSG) Vietnamese About page at `/ve-chung-toi/` (`app/ve-chung-toi/page.tsx`).
* Editorial content in three blocks: site mission, who we are, and how we pick products.
* Restated affiliate disclosure on the page, rendered via the F0005 `<AffiliateDisclosure>`
  component / `AFFILIATE_DISCLOSURE_VI` constant (no copy-pasted literal), with a link to the
  full `/cong-bo-tiep-thi-lien-ket/` page.
* Contact point (email `ttln1201@gmail.com` as a `mailto:` link, matching the footer).
* Page metadata: unique Vietnamese `<title>`, `<meta name="description">`, and canonical URL
  `${NEXT_PUBLIC_SITE_URL}/ve-chung-toi/`.
* Resolving the existing 404 that the header nav and footer "Giới thiệu" links produce.

### Out of Scope
* Team bios, hiring/careers, press kit (per backlog F0010 out-of-scope note).
* Full JSON-LD / Open Graph / breadcrumb SEO treatment and sitemap inclusion — owned by F0009.
  This page gets only the baseline title/description/canonical here (it should still end up in
  the F0009 sitemap when that ships).
* Social-follow links ("Theo dõi") — the footer placeholder stays "Sắp ra mắt"; not introduced
  on the About page in this feature.
* Changing the `/ve-chung-toi/` slug or the nav/footer links that reference it.
* A contact form or any server-side message handling — contact is a static `mailto:` link only.

## 3. Process Flow
1. A visitor clicks "Về chúng tôi" in the header navigation (or "Giới thiệu" in the footer).
2. They land on `/ve-chung-toi/` and receive a `200` (previously a 404).
3. They read the three editorial blocks: the site's mission, who we are, and how we pick
   products.
4. They see the affiliate disclosure restated, with a link to the full disclosure page.
5. They find a contact point (email) to reach the site.

## 4. User Stories & Acceptance Criteria

### US00101: About page route, shell & editorial content
**As a** site visitor,
**I want to** read an About page at `/ve-chung-toi/`,
**So that** I understand who runs the site and how products are chosen, and trust its
recommendations.

**Acceptance Criteria:**
* **Scenario 1:** Page renders (happy path)
    * `Given` the site is built and deployed
    * `When` I navigate to `/ve-chung-toi/`
    * `Then` a statically pre-rendered Vietnamese About page loads with a single `<h1>`,
      semantic body content, and the global `<Header />` / `<Footer />` chrome.

* **Scenario 2:** Header nav and footer links are no longer broken
    * `Given` the header nav link "Về chúng tôi" and the footer link "Giới thiệu", both pointing
      at `/ve-chung-toi/`
    * `When` I click either from any page
    * `Then` I land on the About page and receive a `200` (no `404` / `not-found`).

* **Scenario 3:** Required editorial content present
    * `Given` I am reading the About page
    * `When` I review its sections
    * `Then` it presents three blocks of real Vietnamese copy (no lorem ipsum): (1) the site's
      **mission**, (2) **who we are** ("Chúng tôi là ai"), and (3) **how we pick products**
      ("Cách chúng tôi chọn sản phẩm").

* **Scenario 4:** Conventions
    * `Given` project conventions
    * `When` the page is added
    * `Then` it lives at `app/ve-chung-toi/page.tsx` as a static Server Component (no
      `"use client"`), uses theme tokens and the system font stack (no external fonts), and has
      co-located page-scoped styles following the F0005 policy-page pattern.

### US00102: Restated affiliate disclosure & contact info
**As a** site visitor,
**I want to** see the affiliate disclosure restated and a contact point on the About page,
**So that** I trust the site's monetization is transparent and know how to reach the site.

**Acceptance Criteria:**
* **Scenario 1:** Disclosure reused, not duplicated
    * `Given` the About page body
    * `When` I read the disclosure block
    * `Then` it renders the affiliate disclosure via the F0005 `<AffiliateDisclosure />`
      component / `AFFILIATE_DISCLOSURE_VI` constant (single source of truth, no copy-pasted
      literal) and includes a link to the full `/cong-bo-tiep-thi-lien-ket/` page.

* **Scenario 2:** Contact point present
    * `Given` the About page body
    * `When` I look for how to reach the site
    * `Then` a clearly labelled contact block shows the email `ttln1201@gmail.com` as a
      `mailto:` link, matching the value used in the footer.

* **Scenario 3:** Single source of truth
    * `Given` the disclosure text and contact email are also rendered in the footer
    * `When` the underlying constant/value changes
    * `Then` the About page reflects the change automatically — no second copy of the disclosure
      literal is introduced.

### US00103: About page SEO metadata
**As a** search engine / social crawler,
**I want** the About page to carry a unique title, description, and canonical URL,
**So that** the page is rankable and shareable.

**Acceptance Criteria:**
* **Scenario 1:** Baseline metadata present
    * `Given` the page is built
    * `When` I inspect the document head
    * `Then` it has a unique Vietnamese `<title>` and a `<meta name="description">` that describe
      the About page, exported via the page's `metadata` object.

* **Scenario 2:** Canonical URL
    * `Given` the page is built
    * `When` I inspect the canonical link
    * `Then` it is `${NEXT_PUBLIC_SITE_URL}/ve-chung-toi/` (trailing slash), and a missing
      `NEXT_PUBLIC_SITE_URL` does not crash the build (matches existing project behavior).

* **Scenario 3:** Scope boundary
    * `Given` SEO ownership is split with F0009
    * `When` this story is implemented
    * `Then` it provides only baseline title/description/canonical — JSON-LD, Open Graph,
      breadcrumbs, and sitemap inclusion are deferred to F0009.

## 5. Edge Cases & Error Handling
* **Unknown sub-paths:** `/ve-chung-toi/anything` should 404 normally — this is a leaf static
  route, not a dynamic segment.
* **Missing `NEXT_PUBLIC_SITE_URL`:** Canonical generation must follow the same env-var handling
  used by other pages; a missing value must not crash the build.
* **Empty/placeholder content:** The page must ship with real Vietnamese copy, not lorem ipsum.
* **Disclosure / contact drift:** The disclosure must read from `AFFILIATE_DISCLOSURE_VI` and the
  contact email must match the footer's value; guard against re-introducing a copy-pasted literal
  or a second hard-coded email.
* **Trailing-slash consistency:** The route and canonical URL use the trailing-slash form
  (`/ve-chung-toi/`) to match the header nav and footer links and the site's existing route style.

## 6. Technical Considerations
* **Stack alignment:** SSG only (App Router). The page is a static `app/ve-chung-toi/page.tsx`
  Server Component with a `metadata` export for title/description/canonical.
* **Route & slug (keep Vietnamese, trailing slash):** `/ve-chung-toi/` →
  `app/ve-chung-toi/page.tsx`. This slug is already referenced by the header navigation
  (`lib/nav-items.ts` / `HeaderNav`) and the footer "Giới thiệu" link (`components/Footer.tsx`);
  do not rename it without updating those references in the same PR.
* **Reuse, don't duplicate:** The disclosure flows from `AFFILIATE_DISCLOSURE_VI`
  (`lib/disclosures.ts`) via the F0005 `<AffiliateDisclosure>` component. The contact email
  matches the footer's `CONTACT_EMAIL`.
* **Component conventions:** Static Server Component with co-located page-scoped `.module.css`,
  mirroring the F0005 policy pages (`chinh-sach-bao-mat`, `cong-bo-tiep-thi-lien-ket`). No `lib/`
  JSX.
* **SEO baseline only:** Provide unique title/description/canonical now; defer JSON-LD, Open
  Graph, breadcrumbs, and sitemap inclusion to F0009.
* **Dependencies:** F0002 (header, footer, theme tokens, layout shell) and F0005
  (`<AffiliateDisclosure>` + `AFFILIATE_DISCLOSURE_VI`).
* **Verification:** Per project rules, verify in a browser (header nav + footer "Giới thiệu"
  links resolve to `200`, content renders, single `<h1>`, disclosure + contact present) before
  marking UI stories done.
