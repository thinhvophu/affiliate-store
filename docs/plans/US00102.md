**Status:** Done
**Feature:** F0010 — About Page
**Sibling stories:** US00101 (page shell & editorial content), US00103 (SEO metadata)

---

## 1. Goal

Add two final blocks to the About page (`/ve-chung-toi/`) after the US00101 editorial content:

1. **Restated affiliate disclosure** — rendered via the F0005 `<AffiliateDisclosure />` component (no copy-pasted literal), followed by an inline link to the full `/cong-bo-tiep-thi-lien-ket/` page.
2. **Contact block** — a clearly-labelled Vietnamese section titled `Liên hệ` containing a `mailto:` link to `ttln1201@gmail.com`, sourced from the **same** module the footer uses so that changing the constant propagates everywhere.

The single-source-of-truth guarantee is the load-bearing part: a future change to either the disclosure copy or the contact email must show up automatically on the About page, the footer, and any other surface — no per-page literal.

## 3. Implementation plan

### 3.1 Extract `CONTACT_EMAIL` into `lib/site.ts` (structure-doc impact)

The F0010 spec is explicit: *"the contact email matches the footer's `CONTACT_EMAIL`"* and *"guard against re-introducing a … second hard-coded email"*. Today four call-sites duplicate the literal:

| File | Today | After this PR |
| --- | --- | --- |
| `components/Footer.tsx` | local `const CONTACT_EMAIL = "ttln1201@gmail.com";` | `import { CONTACT_EMAIL } from "@/lib/site";` |
| `app/chinh-sach-bao-mat/page.tsx` | local `const CONTACT_EMAIL = "ttln1201@gmail.com";` | `import { CONTACT_EMAIL } from "@/lib/site";` |
| `app/cong-bo-tiep-thi-lien-ket/page.tsx` | two inline `"ttln1201@gmail.com"` literals (mailto + visible label) | `import { CONTACT_EMAIL } from "@/lib/site";` |
| `app/ve-chung-toi/page.tsx` (new this story) | — | `import { CONTACT_EMAIL } from "@/lib/site";` |

Edit `lib/site.ts` to:

```ts
export const SITE_NAME = "aff-store";
export const CONTACT_EMAIL = "ttln1201@gmail.com";
```

Then migrate the three existing call-sites to the import. This is intentionally bundled into US00102 because that's where the duplicate becomes a triple-source-of-truth bug; the F0010 acceptance Scenario 3 ("Single source of truth") cannot be honestly satisfied otherwise.

### 3.2 Add the disclosure block to `app/ve-chung-toi/page.tsx`

After US00101's three editorial sections (mission / who we are / how we pick products), append a new `<section>` (or `<aside>`-wrapped block) inside the `<article>`:

```tsx
import Link from "next/link";
import { AffiliateDisclosure } from "@/components/AffiliateDisclosure";
// …
<section className={styles.disclosureSection} aria-labelledby="sec-disclosure">
  <h2 id="sec-disclosure" className={styles.sectionHeading}>
    Công bố tiếp thị liên kết
  </h2>
  <AffiliateDisclosure />
  <p className={styles.disclosureMore}>
    Đọc đầy đủ tại{" "}
    <Link href="/cong-bo-tiep-thi-lien-ket/" className={styles.internalLink}>
      Công bố tiếp thị liên kết
    </Link>
    .
  </p>
</section>
```

Notes:

- `<AffiliateDisclosure />` already provides the orange-accent visual band and `aria-label="Công bố tiếp thị liên kết"`. We wrap it in a `<section>` with its own `<h2>` so the page's heading outline reads cleanly — F0005 deliberately keeps the component heading-less.
- The "Đọc đầy đủ tại …" sentence is a *page-level* addition, not a change to the F0005 component. It satisfies the spec's "includes a link to the full `/cong-bo-tiep-thi-lien-ket/` page" requirement without forking the disclosure component.

### 3.3 Add the contact block

Append a second `<section>` to the same `<article>`:

```tsx
import { CONTACT_EMAIL } from "@/lib/site";
// …
<section className={styles.contactSection} aria-labelledby="sec-contact">
  <h2 id="sec-contact" className={styles.sectionHeading}>
    Liên hệ
  </h2>
  <p>
    Mọi câu hỏi, góp ý, hay đề xuất sản phẩm, mời bạn gửi email cho chúng tôi tại{" "}
    <a href={`mailto:${CONTACT_EMAIL}`} className={styles.internalLink}>
      {CONTACT_EMAIL}
    </a>
    .
  </p>
</section>
```

The `mailto:` href is built from the same constant the label renders — no chance of label/href drift.

### 3.4 Add the page-scoped styles

Extend the `page.module.css` US00101 introduces under `app/ve-chung-toi/` with:

- `.sectionHeading` mirroring the F0005 policy-page heading style (left orange border, `--color-primary`).
- `.disclosureSection`, `.contactSection` (just block spacing using `--spacing-xl`).
- `.disclosureMore` — small-meta paragraph, `--color-text-secondary`.
- `.internalLink` — same `--color-primary` underline pattern used in `chinh-sach-bao-mat.module.css` and `cong-bo-tiep-thi-lien-ket/page.module.css`.

No new design tokens are introduced.

### 3.5 Execution order

1. Edit `lib/site.ts` to add `CONTACT_EMAIL`.
2. Migrate `components/Footer.tsx`, `app/chinh-sach-bao-mat/page.tsx`, `app/cong-bo-tiep-thi-lien-ket/page.tsx` to import the constant. Run `grep -rn "ttln1201@gmail" app components lib` and confirm zero matches outside `lib/site.ts`.
3. Append the disclosure + contact sections to `app/ve-chung-toi/page.tsx` (created by US00101).
4. Extend `app/ve-chung-toi/page.module.css` with the new section/link styles.
5. `npm run build` + visual check in the browser (see §6).
