# US00066 — Blog post page `/bai-viet/[slug]/`

**Status:** Done
**Feature:** F0006 (Editorial Content)
**Story:** US00066 — Blog post page `/bai-viet/[slug]/`

---

## 1. Goal

Statically pre-render one blog-post page per known slug at `/bai-viet/[slug]/`. The page
renders, inside the global `<Header />` / `<Footer />` chrome:

1. A **hero image** (from frontmatter `coverImage`, via `next/image`) — omitted gracefully if absent.
2. A single **`<h1>`** = post `title`.
3. A **meta row**: formatted publish date (`formatPostDate()`, US00061) + **byline** (the site name as author).
4. The **`<AffiliateDisclosure>`** banner (US00051) **above** the article body — renders `AFFILIATE_DISCLOSURE_VI`, no copy-pasted literal.
5. The **MDX body** (rendered by US00062's pipeline) below the disclosure.

Routing: `generateStaticParams` pre-renders one page per known slug; unknown slug → `notFound()` →
Vietnamese `not-found.tsx`. `generateMetadata` emits unique Vietnamese `<title>`/`<meta description>`
from frontmatter + canonical `${NEXT_PUBLIC_SITE_URL}/bai-viet/<slug>/` (trailing slash).

**Explicitly NOT in this story** (later F0006 stories / F0009): the TOC (US00068), related posts
(US00067), read-time (US00069), and `Article` JSON-LD / OG / `og:image` / visible breadcrumbs / sitemap
(F0009). The layout must *reserve space* for the TOC + related-posts without building them.

## 3. Dependency boundary with US00062 (MDX body)

`lib/posts.ts` returns `Post.content` as a **raw MDX string** (split out by `gray-matter`), NOT a routed
`.mdx` module. So rendering the body requires a string→React compiler — that compiler + the component
map + heading anchors + `next/image` mapping are **US00062's deliverable**, not this story's.

US00066 must therefore:
- Render the body through whatever entry-point US00062 exposes. The expected seam is a Server Component,
  e.g. `components/PostBody.tsx` taking `{ content: string }` (or a `renderPostBody(content)` helper).
- If US00062 has not landed when US00066 is implemented, ship a **minimal placeholder** body renderer
  (render `post.content` inside a styled `.prose` container as preformatted text) behind the same
  `<PostBody>` seam, with a `// TODO(US00062)` comment. This keeps the page route, metadata, hero,
  disclosure, and 404 fully shippable and verifiable now; US00062 swaps the renderer internals without
  touching `page.tsx`. **Flag to reviewer:** confirm which order US00062/US00066 land in.

## 5. Page composition (JSX shape)

```tsx
return (
  <div className={styles.container}>
    <article className={styles.post}>
      <header className={styles.postHeader}>
        {post.coverImage && (
          <div className={styles.hero}>
            <Image src={post.coverImage} alt={post.title} fill priority
                   sizes="(min-width: 1024px) 760px, 100vw" className={styles.heroImg} />
          </div>
        )}
        <h1 className={styles.title}>{post.title}</h1>
        <p className={styles.meta}>
          <time dateTime={post.publishedAt}>{formatPostDate(post.publishedAt)}</time>
          <span className={styles.byline}>{SITE_NAME}</span>
        </p>
      </header>

      <AffiliateDisclosure />            {/* US00051 — above the body, compliance */}

      <div className={styles.prose}>
        <PostBody content={post.content} />   {/* US00062 seam */}
      </div>

      {/* RESERVED — US00067 related posts mounts here: <RelatedPosts post={post} /> */}
    </article>
  </div>
);
```

- **`SITE_NAME`** byline: use the site name already used by `<Header />` / `<Footer />`. During
  implementation, source it from the same constant the Header uses (do not duplicate a literal); if no
  shared constant exists, add one to `lib/` and note the structure-doc impact. **Open question for reviewer.**
- **Hero graceful degrade:** the `{post.coverImage && …}` guard means a missing/empty `coverImage` simply
  omits the hero frame — no build crash (Edge Case §5). Note: `lib/posts.ts` currently *requires*
  `coverImage` in frontmatter, so in practice it is always present; the guard is belt-and-braces for an
  empty-string or future-optional case.
- `next/image` for the hero satisfies the SEO image rule. Use `fill` + a fixed-ratio `.hero` wrapper.

## 6. Layout — reserving space for TOC (US00068) + related posts (US00067)

The article reads in a **single reading-width column now**, but the layout must let the **left-panel TOC**
slot in later **without rework**. Two viable approaches — recommend **(A)** and flag for reviewer:

- **(A) `<ShellLayout>` from the start (recommended).** Wrap the page in `<ShellLayout leftPanel={…}>`.
  In US00066 pass an **empty/placeholder left panel** (or `leftPanel={null}` with CSS collapsing the
  column to reading width). US00068 then fills `leftPanel` with `<PostToc headings={…} />` and the
  sticky behavior — `page.tsx` barely changes. This matches CLAUDE.md UI rule "Left panel = … TOC (blog
  post)" and F0006 §6 "`<ShellLayout>` / `<Drawer>` for the left panel + mobile collapse". Trade-off:
  `<ShellLayout>` currently always renders the drawer trigger; with an empty panel that trigger should be
  suppressed — small CSS/prop tweak, note as a US00068 follow-up.
- **(B) Plain reading-width container now, refactor to ShellLayout in US00068.** Simpler now, but US00068
  must restructure `page.tsx` — the "without rework" requirement argues against it.

**Decision in this draft: (A).** The CSS module defines a reading-width main column (`max-width ~760px`)
and a left-panel slot sized ~280px (per UI rule) that is empty in US00066. `prefers-reduced-motion`
respected for any future sticky/scroll. **Related posts (US00067)** mount as a sibling `<section>` after
the `.prose` block — leave a clearly-commented placeholder so it appends with zero structural change.

`post-detail.module.css` provides: `.container` (max-width + padding, like product detail), `.hero`
(aspect-ratio framed wrapper for `next/image fill`), `.postHeader`, `.title` (single h1 type scale),
`.meta` / `.byline`, `.prose` (typographic styles for the rendered MDX body — headings, lists, tables,
code, inline images; coordinate token usage with US00062 so the two don't fight over `.prose`).

> Note: if US00062 owns the `.prose` typography, US00066's CSS should only own page chrome (container,
> hero, header, meta) and leave body element styling to US00062. **Flag boundary to reviewer.**
