## 1. Story & Goal

> **As a** content reader, **I want to** see post dates in Vietnamese long form
> (`02 tháng 5, 2026`), **so that** dates read naturally and consistently across the blog.

Deliver `formatPostDate()` as a **new named export in `lib/format.ts`**, sitting beside the existing
`formatVnd()`. It converts an ISO date string (`"2026-05-02"`) into the canonical Vietnamese long
form **`"02 tháng 5, 2026"`** — zero-padded day, the literal word `tháng`, the numeric month (no
leading zero), a comma, then the full year.

This story ships **only the formatter and its chokepoint contract**. No blog surface consumes it yet
— the listing item (US00064), post header (US00066), and related-post card (US00067) will call it
when they ship. This is the date analogue of US00041's `formatVnd` chokepoint.

## 4. Implementation design

### 4.1 Signature & behavior

```ts
/**
 * Render an ISO date string as the canonical Vietnamese long-form date,
 * `"02 tháng 5, 2026"` (zero-padded day, "tháng" + numeric month, full year).
 *
 * SSG determinism (same rationale as formatVnd): `toLocaleDateString("vi-VN")`
 * / `Intl.DateTimeFormat` are intentionally NOT used — ICU month-name / spacing
 * data can drift between Node builds and make HTML non-deterministic across
 * Vercel build pools. Vietnamese months are purely numeric ("tháng 5"), so the
 * output is assembled by hand and is byte-identical on every machine.
 */
export function formatPostDate(iso: string): string
```

**Parsing decision — parse the ISO string by hand, not via `new Date()`.** `new Date("2026-05-02")`
is treated as UTC midnight and then read back through local-timezone getters (`getDate()` etc.),
which can shift the day by one depending on the build machine's `TZ`. To stay deterministic and
timezone-independent (the same SSG concern as `formatVnd`), split the leading `YYYY-MM-DD` portion of
the string directly:

```ts
const match = /^(\d{4})-(\d{2})-(\d{2})/.exec(iso.trim());
```

- `match[1]` → year (4 digits, used verbatim).
- `match[2]` → month: parse to int, render **without** leading zero (`5`, not `05`) → `tháng 5`.
- `match[3]` → day: render **zero-padded to 2 digits** (`02`) → matches the spec example `02`.
- Assemble: `` `${day} tháng ${month}, ${year}` `` → `"02 tháng 5, 2026"`.

This handles the full ISO datetime form too (`"2026-05-02T00:00:00Z"`) since the regex only anchors
the date prefix — `publishedAt` is documented as `"2026-05-01"` but the prefix match is robust either
way.

### 4.2 Edge cases / sentinels (belt-and-braces, mirroring formatVnd)

- **Malformed / non-matching input** (empty string, garbage, no `YYYY-MM-DD` prefix): return `""`
  (empty string) rather than throwing or rendering `NaN`/`Invalid Date`. `lib/posts.ts` (F0001)
  already validates `publishedAt` at build time, so this is a defensive sentinel that should never
  surface in shipped HTML — consistent with the `formatVnd` "should never appear" comment.
- **Month with leading zero in source** (`"05"`): rendered as numeric `5` (strip leading zero).
- **Day already two digits** (`"12"`): rendered as `12` unchanged.
- Optional whitespace around the input is trimmed before matching.

### 4.3 Single-chokepoint enforcement (Scenario 2)

No code change beyond adding the function — enforcement is a **documented convention** consumers must

## 5. Testing / verification

This is a non-UI pure function, so verification is assertion-based rather than browser-based:

1. **Canonical case (Scenario 1):** `formatPostDate("2026-05-02") === "02 tháng 5, 2026"`.
2. **Numeric-month, no leading zero:** `formatPostDate("2026-11-09") === "09 tháng 11, 2026"`.
3. **Datetime prefix tolerated:** `formatPostDate("2026-05-02T13:00:00Z") === "02 tháng 5, 2026"`.
4. **Timezone independence (Scenario 3):** running the same call under `TZ=UTC`, `TZ=America/Los_Angeles`,
   and `TZ=Asia/Ho_Chi_Minh` yields identical output (proves no `new Date()` local-getter drift).
5. **Malformed sentinel:** `formatPostDate("") === ""` and `formatPostDate("not-a-date") === ""`.
6. `tsc --noEmit` + project lint pass; `next build` stays green (no new build-time call sites yet).

If the repo has a test runner wired, add the assertions there; otherwise verify via a throwaway
`node -e` / `tsx` snippet across the three `TZ` values, then discard it (no scratch file shipped).

## 7. Acceptance criteria (verbatim from F0006 §US00061)

1. **Canonical format** — `formatPostDate("2026-05-02")` returns `"02 tháng 5, 2026"` (zero-padded
   day, `tháng` + numeric month, full year).
2. **Single chokepoint** — any blog surface (listing item, post header, related-post card) renders a
   date via `formatPostDate()`; no file outside `lib/format.ts` hand-rolls date formatting or calls
   `toLocaleDateString` / `Intl.DateTimeFormat` on a post date.
3. **Build-environment stability** — the formatter does not rely on locale-dependent runtime ICU
   output that could drift between environments (same rationale as `formatVnd`); the Vietnamese form
   is produced deterministically.
