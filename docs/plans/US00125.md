# Implementation Plan — US00125: Curated manual input for higher-consideration categories

**Status:** Draft
**Feature:** F0012 — Automated Content Ingestion Pipeline
**Spec ref:** `docs/specs/F0012.md` § US00125
**Predecessors:** **US00121** (candidate model + validation engine). Independent of US00124 (the two adapters are siblings that normalize into the same model).
**Successors:** none direct.

---

## 1. Scope

Strictly:

1. Add the **file source adapter** (`scripts/ingest/sources/file.ts`) for `--source=file --path=<file>`.
2. Parse a human-curated candidate list where each entry supplies at minimum a **product name** and a **Shopee URL**, plus optional fields the curator can hand-fill (`brand`, `price`, `description`, `specs`, extra image URLs, `notes`).
3. Normalize each entry into the **shared `Candidate` model** (US00121), so it goes through **identical** validation → staging (US00123) → dedupe (US00122) → writer as the scrape path (Scenario 1).
4. **Malformed input is fatal, pre-ingest** (Scenario 2): a missing required column/field (e.g. no product URL for an entry) exits non-zero, names the offending line/entry, and ingests nothing from that file.

**Explicitly out of scope:**

- **Which** products to include — that is the curator's judgment; the adapter only reads what it's given (spec §2 "Out of Scope").
- Image download → **US00123**. Slug/dedupe → **US00122**.
- Fetching missing fields from Shopee — the curator supplies them, or the candidate is rejected downstream by US00121's engine (closed `Product` shape).

---

## 2. Pre-conditions / current state

- US00121 merged: `Candidate` model + `validateCandidate` + `loadCandidates(args)` dispatch seam + reporter.
- The candidate model requires: `name, brand, price, description, specs, imageUrls[], affiliateUrl, category`. A file entry that only gives name+URL will **validate-reject** (missing brand/price/specs/images) unless the curator fills those in — so the input format must let curators supply the full set. This plan uses a format that carries all `Product` content fields; the **only hard-required columns are `name` + `url`** (so a sparse entry fails validation *with a clear per-candidate reason*, not a parse error), while a **structurally malformed entry** (e.g. a row with no `url` key at all) is a **fatal parse error** (Scenario 2).

> Distinction that drives the design:
> - **Structural/parse error** (unreadable file, an entry missing the `url` field entirely, invalid JSON/format) → **fatal**, exit non-zero, name the line (Scenario 2).
> - **Content-incomplete but well-formed** (has `url` + `name` but no `price`) → **per-candidate rejection** via US00121's engine (Scenario 1's "identical validation").

## 3. Approved Decisions

| # | Decision | Choice | Rationale |
| --- | --- | --- | --- |
| D1 | Input file format | **JSON array** (`.json`) of entry objects; each entry `{ name, url, brand?, price?, description?, specs?, images?, notes? }` | JSON round-trips `specs` (a `Record<string,string>`) and Vietnamese diacritics without CSV-escaping pain. A CSV variant is a future add if operators prefer spreadsheets. |
| D2 | Hard-required columns | `name` + `url` only | Matches spec Scenario 2 ("no product URL" is the canonical malformed case). Everything else, if absent, becomes a **content rejection**, not a parse error. |
| D3 | `url` → `affiliateUrl` | Passed through untouched to the candidate; validated by `assertAffiliateUrl` in US00121 | No file outside `lib/affiliate.ts` may parse/trim/rewrite an affiliate URL — the adapter must **not** normalize it. |
| D4 | `category` | Injected from `--category` (not read from the file) | Same as scrape (US00124 D7); the operator declares the target category, registry validates it. |
| D5 | Malformed = fatal, pre-ingest | If any entry lacks `url` or `name`, or the file isn't a JSON array, **exit non-zero naming the entry index/line before ingesting any entry** | Scenario 2: "before ingesting any entry from that file." A two-pass approach: pass 1 validates structure of *all* entries (collect fatal errors); if any, print + exit. Pass 2 ingests. |
| D6 | Reuse validation | File candidates go through `validateCandidate` unchanged | Scenario 1: "identical validation, image staging, dedupe, and fixture output as the scrape path." |
| D7 | `notes` field | Read but **not** written to the fixture (not a `Product` field); may be echoed in the summary for the curator's reference | Keeps curator annotations out of the closed `Product` shape. |
| D8 | Line/entry identification | `sourceRef = "file:<path>#<index>"` (0-based entry index; JSON has no line numbers, so index is the stable locator) | Gives the operator a precise pointer in both fatal and per-candidate messages. If a line number is desired, parse with a location-aware JSON reader — deferred unless operators ask. |

### 4.4 Adapter sketch

```ts
// scripts/ingest/sources/file.ts
export async function loadFileCandidates(args: IngestArgs): Promise<Candidate[]> {
  const raw = readFileOrFatal(args.path);                 // fatal if unreadable
  const entries = parseJsonArrayOrFatal(raw, args.path);  // fatal if not an array
  // Pass 1 — structural validation across ALL entries, before ingesting any (Scenario 2)
  const structuralErrors = entries.flatMap((e, i) => structuralProblems(e, i));
  if (structuralErrors.length)
    throw new FatalIngestError(
      `file source ${args.path}: malformed entries:\n` + structuralErrors.join("\n"),
    );
  // Pass 2 — normalize
  return entries.map((e, i) => ({
    name: e.name, brand: e.brand ?? "", price: e.price ?? NaN,
    affiliateUrl: e.url, description: e.description ?? "",
    specs: e.specs ?? {}, imageUrls: e.images ?? [],
    category: args.category, sourceRef: `file:${args.path}#${i}`,
  }));
}
```

> Note: pass 2 leaves incomplete-but-well-formed entries with empty/NaN fields so US00121's `validateCandidate` produces the **per-candidate** rejection (with the field named). Only *structural* problems are fatal in pass 1.

## 9. Risks / open items

- **CSV request.** Operators used to spreadsheets may want CSV. Deferred; JSON chosen for `specs`/diacritics fidelity (D1). Revisit if requested — the adapter's parse layer is swappable behind `loadFileCandidates`.
- **Curator supplies bad `specs` type** (e.g. an array). Guard in `structuralProblems` or let `validateCandidate` reject; ensure the reason is clear either way.
- **Relative `--path`** resolved against `process.cwd()`; document that in README to avoid confusion.
- **Duplicate entries within one file** (same URL twice) — handled by US00122 dedupe (second is skipped as duplicate); note the interaction in README.
