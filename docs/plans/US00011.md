## 1. Scope

This story is **strictly** about getting a Next.js (App Router) + TypeScript project to:

1. Boot in development.
2. Build for production with SSG output.
3. Enforce TypeScript strict mode.
4. Pin a Node version.

**Out of scope** (owned by sibling stories in F0001):

- Folder structure & path aliases — **US00012**
- MDX pipeline — **US00013**

- Content loaders — **US00015**
- Global styles & tokens — **US00016**
- ESLint + Prettier — **US00017**
- Typed env var module — **US00018**

Do not pre-empt these stories from US00011 work.

## 3. Approved Decisions

| #   | Decision                | Choice                                                                                | Rationale                                                                                                                                  |
| --- | ----------------------- | ------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| D1  | Initialize git locally? | **Yes** — `git init`, default branch `main`, initial commit. No remote (F0011's job). | Slash-command workflow + CLAUDE.md assume `main` branch.                                                                                   |
| D2  | Package manager         | **npm**                                                                               | Simplest baseline; no extra tooling.                                                                                                       |
| D3  | Bootstrap method        | **Manual scaffold** (write files directly) — _not_ `create-next-app`                  | `create-next-app` pulls Tailwind / `src/` / opinionated ESLint that conflict with US00012/US00016/US00017.                                 |
| D4  | SSG mode                | **Default Next.js SSG** (no `output: 'export'`)                                       | Hosted on Vercel, which pre-renders App Router routes at build. `output: 'export'` would block `next/image` default loader needed by spec. |
| D5  | Node version pin        | **Both `.nvmrc` (`22`) and `engines.node` (`>=22.11.0`) in `package.json`**           | Covers nvm users _and_ CI/Vercel. AC requires only one; pinning both is cheap insurance.                                                   |
| D6  | Next.js version         | **`next@15` (latest stable)** with **React 19**                                       | Mature App Router, current LTS line.                                                                                                       |
| D7  | TS strictness           | **Just `strict: true`**                                                               | Don't pre-empt US00017's lint/format polish.                                                                                               |

### 6.2 `tsconfig.json` essentials

- `"strict": true` (AC Scenario 3 — non-negotiable)
- `"jsx": "preserve"`

- `"target": "ES2022"`, `"lib": ["dom", "dom.iterable", "es2022"]`

- `"include": ["next-env.d.ts", "**/*.ts", "**/*.tsx", ".next/types/**/*.ts"]`
- `"exclude": ["node_modules"]`
- **No `paths` aliases yet** — US00012 owns those.

## 8. Acceptance-Criteria Verification

| AC                               | Test                                                                                                | Pass condition                                                          |
| -------------------------------- | --------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------- |
| Scenario 1 — boots in dev        | `npm run dev`, `curl -I http://localhost:3000`                                                      | HTTP 200; no Pages Router warning in stdout.                            |
| Scenario 2 — production build    | `npm run build`                                                                                     | Exit 0; build output shows `/` as `○ (Static)`; `.next/` produced.      |
| Scenario 3 — TS strict enforced  | Temporarily inject `const x: number = "bad";` into `app/page.tsx`; run `npm run typecheck`; revert. | Non-zero exit with `TS2322`. After revert, `npm run typecheck` exits 0. |
| Scenario 4 — Node version pinned | `cat .nvmrc` and `node -p "require('./package.json').engines.node"`                                 | Both return non-empty values (`22` and `>=22.11.0`).                    |

Repeat the loop until all four pass.

## 9. Risks / Open Items

- **No git remote yet.** This story commits locally only. Pushing waits for F0011 (Vercel + GitHub remote setup).
- **Node 24 vs pinned 22.** Local machine is on Node 24; Next.js 15 supports both. Pinning to 22 is for consistency and matches Vercel's current LTS default. Not a blocker.
- **Placeholder `app/page.tsx`** will be replaced wholesale by F0009 (Homepage). Keep it intentionally minimal so the diff is clean.
- **`next-env.d.ts`** is regenerated by Next.js; we commit it (Next.js convention) so fresh clones type-check before the first `next` invocation.
