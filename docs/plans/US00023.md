# Implementation Plan — US00023: Root layout — shell wiring + Shopee surface tokens

**Status:** Approved
**Feature:** F0002 — Site Shell & Theme
**Spec ref:** `docs/specs/F0002.md` § US00023
**Predecessors (hard):** US00021 (Site Header component), US00022 (Site Footer component). This story wires both into the root layout — it cannot land before either ships, and is best landed in the same PR train as US00021/US00022.
**Predecessors (preserved):** US00016 (base tokens + globals.css), US00018 (typed env import), US00115 (`<SpeedInsights />` mount).
**Successors:** US00024 (two-column shell layout) and US00026 (sticky header P2) consume invariants set by this story — `<main>` landmark, surface tokens, body page-bg.

---

## 1. Scope

Strictly:

1. Extend `app/layout.tsx` so every route's content is wrapped in `<Header /> + <main>{children}</main> + <Footer />`, while **preserving every existing invariant** (`<html lang="vi">`, the `@/lib/env` side-effect import, the `./globals.css` import, and the `<SpeedInsights />` mount).
2. Insert a Vietnamese-labelled **skip-to-main-content** link as the first focusable element inside `<body>`. The link must be visually hidden by default, become visible on first Tab, jump focus to `<main>` when activated, and meet WCAG AA contrast against any backdrop it can appear in front of (notably the orange Header band).
3. Make `<main>` the single canonical landmark on every page (`id="main-content"`, `tabIndex={-1}` so the skip link can move focus into it programmatically).
4. **Additively** extend `app/globals.css` with six F0002 Shopee surface tokens on `:root` — without redefining or shadowing any token US00016 already declared:
   - `--color-page-bg: #F5F5F5`
   - `--color-surface: #FFFFFF`
   - `--color-border-hairline: #E5E5E5`
   - `--color-on-primary: #FFFFFF`
   - `--radius-sm: 2px`
   - `--radius-md: 4px`
5. Repoint `<body>`'s `background-color` from `var(--color-bg)` (white, US00016) to `var(--color-page-bg)` (light gray, F0002) so content cards visually float on a Shopee-style page surface.
6. Confirm all new tokens remain overridable via standard CSS scoping (any descendant rule that re-sets the variable wins for its subtree).

**Explicitly out of scope** (owned by sibling F0002 stories or other features):

- Header component itself (markup, nav links, mobile hamburger trigger) — **US00021**.
- Footer component itself (multi-column layout, disclosure block, policy links, year) — **US00022**.
- Two-column shell layout component (`leftPanel` + `children` slot API) — **US00024**.
- Mobile drawer (slide-out, focus trap, scroll lock, overlay) — **US00025**.
- Sticky-on-scroll Header behavior + drop shadow — **US00026** (P2).
- Privacy Policy / Affiliate Disclosure destination pages — **F0005**.
- Concrete page bodies (homepage hero, product list/detail, blog) — F0004/F0006/F0008.
- Any redefinition of US00016 tokens (`--color-primary`, `--color-text`, `--color-bg`, breakpoints, spacing). This story is **additive only**.
- Deprecation/removal of US00016's `--color-bg` and `--color-bg-secondary` — see § 10 risk; tracked as a follow-up, not done here.

### 5.5 `app/globals.css` — skip-link rules (append to end of file)

```css
/* --------------------------------------------------------------------------
   3. Skip link (US00023 — visually hidden until focused)
   -------------------------------------------------------------------------- */

.skip-link {
  position: absolute;
  top: 0;
  left: 0;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0 0 0 0);
  white-space: nowrap;
  border: 0;
}

.skip-link:focus,
.skip-link:focus-visible {
  position: fixed;
  top: var(--spacing-sm);
  left: var(--spacing-sm);
  width: auto;
  height: auto;
  padding: var(--spacing-sm) var(--spacing-md);
  margin: 0;
  overflow: visible;
  clip: auto;
  background-color: var(--color-primary);
  color: var(--color-on-primary);
  border-radius: var(--radius-sm);
  outline: 2px solid var(--color-on-primary);
  outline-offset: 2px;
  z-index: 1000;
  text-decoration: underline;
}

/* Programmatic focus on <main> (after skip-link activation) should not draw a focus ring. */
main:focus {
  outline: none;
}
```

Notes:
- Uses the legacy `clip` property + the modern visually-hidden pattern; both browsers honor it. `clip-path: inset(50%)` is a more modern alternative — choose at implementation time whichever the team prefers, but document the choice.
- `main:focus { outline: none }` prevents a visible focus ring on the `<main>` element after the skip link transfers focus (the page itself is the target, not an interactive control). Page-internal focusable elements still get their normal ring.
- All color/spacing/radius values flow through tokens — no raw hex outside the `:root` block (per spec Edge-Case "Token drift").

## 10. Risks / open items

- **Naming overlap with US00016 tokens.** `--color-bg-secondary` (US00016, `#f5f5f5`) and `--color-page-bg` (F0002, `#F5F5F5`) hold the **same hex value** but are semantically distinct. Likewise `--color-border` (#e0e0e0) and `--color-border-hairline` (#e5e5e5) are close in value but named for different purposes. Keeping both is intentional (D8) to avoid breaking F0001 consumers. **Follow-up:** once F0004/F0006 land their grids and confirm which token they prefer, file a cleanup story to consolidate (e.g., alias `--color-bg-secondary` → `var(--color-page-bg)` and remove the duplicate). Do **not** do this consolidation in US00023.
- **Hard dependency on US00021 / US00022.** If `components/Header.tsx` or `components/Footer.tsx` does not exist when this story is implemented, `app/layout.tsx` will fail to compile. The implementer must verify both are present (or land them in the same PR train). Pause and report rather than stubbing the components.
- **Export shape locked in.** Per the F0002 decisions walkthrough, all component files use **named exports** (only `app/` pages/layouts use default exports per Next.js requirement). `app/layout.tsx`'s own `export default function RootLayout` stays — that's the page-level requirement.
- **Page files that already render their own `<main>`.** None today (only `app/page.tsx` exists and renders a bare `<h1>`). If a future page file accidentally re-introduces `<main>`, ESLint won't catch the double-landmark. Mitigation: a brief PR-review checklist note in the F0002 closure PR description.
- **Skip-link focus styling on non-orange surfaces.** The skip link is positioned over the Header band (orange). If a future story ever renders a non-orange element at the top of the viewport (unlikely, but theoretically possible if the Header is briefly hidden), contrast may regress. The 2px outline mitigates this — the link is visible against either background.
- **`tabIndex={-1}` on `<main>` and screen readers.** Some screen readers can be tripped up by focusable non-interactive landmarks. The widely-tested skip-link pattern is exactly this (`tabIndex={-1}` + `id` target), so risk is low. Verify with VoiceOver (macOS) during browser smoke if available.
- **Shared `body` background regression.** Pages that currently assume `body { background: white }` may visually break (e.g., a content area with no white surface of its own will now look "transparent" over the gray page-bg). The only existing page is `app/page.tsx` with a single `<h1>` — acceptable to leave looking minimal until F0008 ships the homepage. Document in PR description.
- **Skip-link `href="#main-content"` and Next.js routing.** Next.js intercepts in-page `<a>` clicks. Hash-only anchors (`#main-content`) are not treated as route changes — they trigger native browser hash navigation + focus move, which is what we want. Verified per Next.js App Router default behavior. If a future regression changes this, fall back to a `"use client"` skip-link with an `onClick` handler that calls `document.getElementById('main-content')?.focus()`.
- **CSS specificity for `body` rule.** The change from `var(--color-bg)` to `var(--color-page-bg)` is in the same rule — no cascade conflict. If a future story overrides body background, follow the standard `:root` token override route (D8 explicitly leaves this open).
