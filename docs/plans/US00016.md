# Implementation Plan -- US00016: Base global styles & design tokens

**Status:** Approved
**Feature:** F0001 -- Technical Foundation & Bootstrap
**Spec ref:** `docs/specs/F0001.md` section US00016
**Predecessors:** US00011 (Next.js + TS bootstrap), US00012 (canonical folders + `@/*` alias)

---

## 1. Scope

Strictly:

1. Create a global CSS file (`app/globals.css`) containing a CSS reset, CSS custom-property design tokens, and the system font stack.
2. Import that file once in the root layout (`app/layout.tsx`) so every page inherits it.
3. Define `--color-primary: #EE4D2D` as a CSS variable on `:root`, overridable via standard CSS scoping.
4. Define breakpoint tokens as documented CSS custom properties (768px tablet, 1280px desktop).
5. Enforce mobile-first approach (base styles target mobile; media queries use `min-width`).
6. Use system font stack only -- no external font CDN requests.

**Explicitly out of scope** (owned by sibling stories or downstream features):

- Shared TS types -- **US00014**
- Content loaders -- **US00015**
- ESLint / Prettier -- **US00017**
- Typed env vars -- **US00018**
- Header / Footer / Site shell -- **F0006**
- Real page layouts, grids, typography hierarchy beyond reset -- downstream features
- Dark mode -- out of scope for v1

### 5.1 `app/globals.css`

```css
/* ==========================================================================
   Global Styles & Design Tokens
   US00016 — Base global styles & design tokens
   ========================================================================== */

/* --------------------------------------------------------------------------
   1. Design Tokens (CSS Custom Properties)
   -------------------------------------------------------------------------- */

:root {
  /* -- Colors -- */
  --color-primary: #ee4d2d;
  --color-text: #222222;
  --color-text-secondary: #666666;
  --color-bg: #ffffff;
  --color-bg-secondary: #f5f5f5;
  --color-border: #e0e0e0;

  /* -- Typography -- */
  --font-family-base:
    -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, "Noto Sans",
    sans-serif, "Apple Color Emoji", "Segoe UI Emoji", "Segoe UI Symbol", "Noto Color Emoji";
  --font-size-base: 1rem; /* 16px */
  --line-height-base: 1.6;

  /* -- Spacing scale -- */
  --spacing-xs: 0.25rem; /* 4px  */
  --spacing-sm: 0.5rem; /* 8px  */
  --spacing-md: 1rem; /* 16px */
  --spacing-lg: 1.5rem; /* 24px */
  --spacing-xl: 2rem; /* 32px */
  --spacing-2xl: 3rem; /* 48px */

  /* -- Breakpoints (reference only — cannot be used inside @media queries)
     Use these raw values in media queries:
       @media (min-width: 768px)  { ... }   tablet
       @media (min-width: 1280px) { ... }   desktop
  -- */
  --breakpoint-tablet: 768px;
  --breakpoint-desktop: 1280px;

  /* -- Layout -- */
  --max-width: 1280px;
  --sidebar-width: 280px;
}

/* --------------------------------------------------------------------------

  background-color: var(--color-bg);

  -moz-osx-font-smoothing: grayscale;
```

## 10. Risks / open items

- **CSS custom properties in `@media` queries.** CSS spec does not allow `var()` inside `@media` conditions. The breakpoint tokens (`--breakpoint-tablet`, `--breakpoint-desktop`) are declared for documentation/JS access purposes. Components must use the raw `768px` / `1280px` values in their `@media` rules. This is a known CSS limitation, not a bug. The comment in `globals.css` makes this explicit.
- **Reset aggressiveness.** The reset zeroes heading sizes and list styles. Downstream stories (F0006 typography, F0003 blog post styling) will need to add their own heading scale and list styles. This is intentional -- the reset provides a clean slate.
- **No `@font-face` declarations.** System font stack only. If a custom font is ever needed (out of scope per v1), it would be added here.
- **Token set is intentionally minimal.** Only tokens referenced by the spec or CLAUDE.md are included. Downstream features may extend `:root` with additional tokens (e.g., `--color-success`, `--radius-*`), but they should add to `globals.css` rather than creating new global CSS files.
