# Implementation Plan — US00031: Canonical `<AffiliateLink>` primitive

**Status:** Approved
**Feature:** F0003 — Affiliate Link Routing
**Spec ref:** `docs/specs/F0003.md` § US00031
**Predecessors / siblings:**

- **First story in F0003.** No upstream story depends on, or has shipped, the primitive yet. Nothing in the existing tree imports `@/components/AffiliateLink` or `@/lib/affiliate`.
- **Sibling US00032 (whole-card clickability)** is being planned in parallel and consumes the primitive that this story ships. US00031 must accept arbitrary `children` and emit a single `<a>` with no wrapper that breaks card layouts, but **does not** ship the whole-card example or focus-ring CSS — those belong to US00032.
- **Sibling US00033 (tracking seam) is folded into this PR.** Per approved decision, US00031 and US00033 ship together on `feat/US00031`. US00031 emits the four `data-*` attributes _and_ owns the contract documentation (component-file JSDoc block + a new "Affiliate links" section in `CLAUDE.md` that names the four attributes as a public API). The two stories are inseparable in implementation and a single PR is the review gate for both.
- **Sibling US00034 (URL helper + no-raw-anchor convention)** is being planned in parallel and owns `lib/affiliate.ts` (host allow-list, build-time validation hooked into `lib/products.ts`) and the no-raw-anchor convention text in `CLAUDE.md`. US00031 does **not** create `lib/affiliate.ts` or wire validation into `getAllProducts()`; it only consumes the helper if it exists. The primitive's runtime behavior must be correct even if US00034 hasn't landed yet (callers pass already-trusted strings).
- **Downstream consumers (out of this story):** F0004's product card / product detail CTA, F0006's inline MDX `<ProductCard />`, homepage featured picks. All will import `<AffiliateLink>` from `@/components/AffiliateLink` after F0003 lands.

## 1. Scope

Strictly:

1. Create a new Server Component `components/AffiliateLink.tsx` that renders a single `<a>` element with affiliate-correct link semantics, an accessible Vietnamese screen-reader label, and the F0003 ↔ F0007 tracking-seam data attributes.
2. Bake `target="_blank"` and `rel="noopener noreferrer sponsored"` into the component — never accept overrides from the call site.
3. Type the public prop signature so `href`, `productName`, `productCategory`, and `children` are required and non-`any`, and so `target` / `rel` cannot be passed (or are silently ignored).
4. Emit four data attributes on the anchor — `data-affiliate-link`, `data-product-name`, `data-product-category`, `data-destination-url` — populated from props. Names are the public seam contract for F0007.
5. Provide the accessible label `Mở {productName} trên Shopee (mở tab mới)` via a visually hidden `<span>` inside the anchor (preferred over `aria-label` so visible children are not overridden — see D4).
6. Pass `children` through unchanged inside the anchor. The component must not inject layout wrappers (no extra `<div>` / `<span>` around children) that would interfere with whole-card layouts (US00032).
7. Add a minimal CSS Module `components/AffiliateLink.module.css` that contains **only** the visually-hidden helper class for the screen-reader label. No layout, color, padding, hover, or focus rules — the primitive is visually neutral and consumers style children (US00032 extends this for whole-card focus/hover).
8. Include a top-of-file JSDoc that names the usage convention ("All affiliate destinations route through this component") and lists the four seam attributes as the F0003 ↔ F0007 public contract (US00033 content folded in).
9. Add a `className?: string` prop so consumers (notably US00032's whole-card pattern) can attach styles to the wrapping `<a>`. `target`/`rel` remain type-banned via the explicit prop interface.
10. Create a new **"Affiliate links"** section in `CLAUDE.md` that documents (a) the F0003 ↔ F0007 contract attribute set and (b) is the home where US00034's no-raw-anchor convention will append on landing. Update the `Codebase structure` section of `CLAUDE.md` and `README.md` to list the new component / CSS Module, and bump the "Last updated" annotation.

**Explicitly out of scope** (owned by sibling stories or downstream features):

- `lib/affiliate.ts` host allow-list, `assertAffiliateUrl()`, and build-time validation hooked into `getAllProducts()` → **US00034**.
- Whole-card clickability example, focus-visible/hover treatment that wraps an entire card surface, documented copyable usage example for product cards → **US00032**.
- The no-raw-anchor convention bullet in the new `CLAUDE.md` "Affiliate links" section ("Raw `<a href="https://shope.ee/...">` outside `<AffiliateLink>` is disallowed") → **US00034** appends to the section created by this PR.
- The actual delegated click handler that reads `data-affiliate-link` and fires GA4 `affiliate_click` → **F0007**.
- Product card visuals, product detail CTA layout, inline MDX `<ProductCard />` → **F0004 / F0006**.
- Custom ESLint rule enforcing no-raw-anchor → P2 follow-up, deferred.

## 5. Concrete diff sketch

### 5.1 `components/AffiliateLink.tsx`

```tsx
import styles from "./AffiliateLink.module.css";

/**
 * Canonical affiliate-link primitive — F0003 (US00031 + US00033).
 *
 * All affiliate destinations on the site MUST route through this component.
 * The component bakes in correct link semantics, the Vietnamese accessible
 * label, and the F0003 ↔ F0007 tracking-seam contract. See the "Affiliate
 * links" section of CLAUDE.md for the project-wide convention.
 *
 * --- F0003 ↔ F0007 contract (US00033) ---------------------------------
 * Every rendered anchor carries FOUR stable data-* attributes that F0007's
 * delegated click listener relies on. These names are a PUBLIC API of this
 * primitive — renaming any of them is a breaking change and must update the
 * F0007 listener in the same PR.
 *
 *   data-affiliate-link              presence flag (selector hook)
 *   data-product-name                from productName prop
 *   data-product-category            from productCategory prop
 *   data-destination-url             from href prop (mirrors the href)
 *
 * No other component in the codebase may emit `data-affiliate-link`.
 *
 * Until F0007 ships, the seam is INERT: no JS handler is attached from F0003,
 * the click triggers a normal new-tab navigation, and analytics are silently
 * absent. JS-off behaviour is identical (the anchor is a plain server-rendered
 * <a>).
 *
 * Example rendered DOM:
 *
 *   <a href="https://shope.ee/abc"
 *      target="_blank"
 *      rel="noopener noreferrer sponsored"
 *      data-affiliate-link=""
 *      data-product-name="Chuột Logitech G102"
 *      data-product-category="chuot-gaming"
 *      data-destination-url="https://shope.ee/abc">
 *     Mua ngay
 *     <span class="srOnly">Mở Chuột Logitech G102 trên Shopee (mở tab mới)</span>
 *   </a>
 * -----------------------------------------------------------------------
 *
 * Whole-card usage example is in US00032's plan / implementation; consumers
 * import `./AffiliateLink.module.css` and pass `className={styles.card}`.
 * URL validation lives in lib/affiliate.ts (US00034); this component does not
 * re-validate at render time — destinations are already trusted by the loader.
 */

export interface AffiliateLinkProps {
  href: string;
  productName: string;
  productCategory: string;
  children: React.ReactNode;
  /** Optional styling hook applied to the wrapping `<a>`. Used by US00032's
   *  whole-card pattern (consumers import `AffiliateLink.module.css` and pass
   *  `className={affiliateStyles.card}`). target/rel are deliberately not
   *  accepted — link semantics are baked in. */
  className?: string;
}

export function AffiliateLink({
  href,
  productName,
  productCategory,
  children,
  className,
}: AffiliateLinkProps) {
  return (
    <a
      href={href}
      target="_blank"
      rel="noopener noreferrer sponsored"
      // US00033 — F0003 ↔ F0007 contract attributes. Do not rename without
      // updating the F0007 delegated click listener in the same PR.
      data-affiliate-link=""
      data-product-name={productName}
      data-product-category={productCategory}
      data-destination-url={href}
      className={className}
    >
      {children}
      <span className={styles.srOnly}>
        {`Mở ${productName} trên Shopee (mở tab mới)`}
      </span>
    </a>
  );
}
```

## 10. Risks / open items

- **Sibling coordination.** US00032 / US00033 / US00034 are being planned in parallel and all touch the same component file (US00032 + US00033 only documentation, US00034 not at all) or the same `CLAUDE.md` section. If two stories land out of order, the structure-doc rows may collide. Mitigation: each PR's "Last updated" line names its own story; whoever rebases last reconciles the ordering. The component file itself is only physically touched by US00031.
- **Whole-card layout coupling.** US00032 needs to wrap large card subtrees as children. The primitive intentionally does not inject wrapper elements (D11), but the trailing visually-hidden `<span>` is a sibling of the user's children. Confirmed safe: it is absolutely positioned and 1×1, so it does not occupy layout space and does not affect a `display: grid` / `display: flex` parent in `<a>`. If US00032 uncovers a corner case, the fix is to move the hidden span ahead of children or wrap children in a `<>` fragment — both are zero-impact tweaks.
- **`<a>` cannot wrap interactive descendants (`<button>`, nested `<a>`, `<input>`).** The HTML spec disallows it; US00032 documents the rule as part of the whole-card convention. US00031 cannot detect this at compile time without runtime children-traversal (rejected as overkill). Code review + the convention text in US00032 / US00034 are the enforcement mechanism. Spec §5 explicitly accepts this trade-off.
- **`data-product-name` with quotes / special characters.** React escapes attribute values when interpolated; the harness step in §6 exercises `'Bàn phím "ROG" Strix'` to confirm. No mitigation needed beyond the existing React behavior.
- **No URL validation in the primitive.** If a caller passes a non-Shopee URL (or empty string) before US00034 lands, the link still renders and works. Risk window is exactly between US00031 landing and US00034 landing. Mitigation: US00034 is scheduled in the same feature batch (F0003), and the only places that would call the primitive (F0004 / F0006) do not exist yet either. Net effect: no production exposure.
- **`aria-label` vs. visually-hidden span trade-off.** D4 chose the hidden span. If a sibling story or downstream user reports that the announced name is too verbose (visible text + hidden label), the fix is to swap the visible text into the hidden label and use `aria-label` for the canonical accessible name. This is a one-line change inside `<AffiliateLink>` and does not affect call sites.
- **`className` allowed; `target`/`rel`/`style` not.** Per the Q1 design decision, `className?: string` is the single styling escape hatch. `style` is omitted — anyone needing inline styles should use a class. `target` and `rel` are type-banned because they would weaken the affiliate-link contract. If a future consumer needs to set `id` or another DOM attribute, evaluate at that time — the safe pattern is to expose it explicitly (one prop at a time), never to spread `…rest` onto the anchor.
- **Server Component graph integrity.** Adding `<AffiliateLink>` to any page (Server or Client) is safe because it does not import any client-only API and does not declare `"use client"`. Importing it inside a `"use client"` parent will simply opt the rendered tree into the client component graph for that parent — still safe, but discouraged for SSG bundle hygiene.
