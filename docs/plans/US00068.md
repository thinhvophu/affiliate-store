# US00068 — Sticky table of contents (TOC)

**Feature:** F0006 (Editorial Content)
**Status:** Approved
**Priority:** P1
**Depends on:** US00062 (MDX heading anchors), US00066 (post page `/bai-viet/[slug]/` + left-panel shell), US00024/25 (`<ShellLayout>` / `<Drawer>`), US00026 (`HeaderStickyShadow` IntersectionObserver pattern to mirror).

---

## 1. Story & goal

A reader on a long buying-guide post (`/bai-viet/[slug]/`) sees a left-panel table of contents listing the article's `h2`/`h3` sections. Clicking an entry scrolls to / deep-links the section. On desktop the TOC stays sticky while scrolling. On mobile it collapses (drawer/disclosure or hidden). When a post has **no** `h2`/`h3` headings, the TOC is omitted gracefully and the body still reads normally.

## 2. Acceptance criteria (from F0006 §US00068)

- **AC1 — TOC from headings:** desktop left panel lists the post's `h2`/`h3` headings; clicking an entry scrolls to / hash-deep-links the matching section.
- **AC2 — Sticky on desktop:** on a desktop viewport with a post taller than the viewport, the TOC stays visible (sticky) in the left panel while the article scrolls.
- **AC3 — Mobile / no-heading behavior:** TOC collapses appropriately on mobile (drawer/disclosure or hidden) and is omitted entirely when there are no `h2`/`h3` headings — the body still reads normally (F0006 §5 "Post with no headings").

## 4. Heading extraction — design contract (locked with US00062)

`Post.content` is the raw MDX body string (`lib/posts.ts`, `parsed.content`). US00062 introduces
the single heading-slug chokepoint at **`lib/mdx-slug.ts`**, exporting
`createHeadingSlugger(): { slug(text): string }` (a wrapper around `github-slugger` with a fresh
instance per document) plus the small custom rehype plugin `rehypeHeadingSlugs` that US00062's
`PostBody` calls inside `evaluate()`. **No `rehype-slug`** dependency — the chokepoint owns it.

**Approach (build-time extraction in `lib/`):** add `lib/toc.ts` exporting
`extractToc(content: string): TocEntry[]` where:

```ts
export interface TocEntry { depth: 2 | 3; text: string; id: string; }
```

It parses the MDX/markdown body for `h2`/`h3` headings via an AST walk and computes each `id` by
calling `createHeadingSlugger()` from `lib/mdx-slug.ts` in heading order. **Same function, same
input order = byte-identical ids** to what `PostBody`'s `rehypeHeadingSlugs` stamps on the
rendered headings. The page calls `extractToc(post.content)` at build time and passes the result
to `<TableOfContents>` — no client parsing, no JSX in `lib/`.

**Parsing note:** prefer an AST pass (`remark-parse` → `mdast` `visit` on `heading` nodes,
extracting depth + plain text) over a regex, so fenced code blocks containing `## …` are not
mistaken for headings. `remark` is already a transitive dep of `@next/mdx`; confirm a direct dep
(`remark-parse`/`unist-util-visit`) is added if needed, and read `node_modules/next/dist/docs/`
for the current MDX guidance per `AGENTS.md`.

> Convention to add (US00062 also documents this): *"Heading slugs for MDX bodies come from
> `lib/mdx-slug.ts`. No other file may call `github-slugger` or hand-roll heading slugs."* US00068
> consumes the chokepoint; it does not duplicate the slug logic.

## 5. Component & integration design

- **`components/TableOfContents.tsx`** — Server Component. Props: `entries: TocEntry[]`. Renders `<nav aria-label="Mục lục">` with a nested list; each item is `<a href={"#" + id}>` (native hash anchor → free deep-link + scroll, no JS required for AC1). `h3` entries indented under their preceding `h2`. **If `entries.length === 0`, render `null`** (AC3 no-heading omission).
- **`components/TableOfContents.module.css`** — sticky positioning (`position: sticky; top: <header-height + gap>`), `max-height: calc(100vh - …); overflow: auto`, token-based type, `--color-primary` accent for the active/hover link. Respect `prefers-reduced-motion` for any smooth-scroll.
- **Smooth scroll & deep-link:** rely on CSS `scroll-behavior: smooth` (gated by `prefers-reduced-motion: no-preference`) and `scroll-margin-top` on `h2`/`h3` so the sticky header does not overlap the target. Native `href="#id"` handles scroll + URL hash with zero client JS.
- **Active-section highlight — deferred** (C4). The AC does not mandate scroll-spy; this story
  ships AC1–AC3 only. No `TableOfContentsClient.tsx` in scope. Revisit as a follow-up only if a
  user-facing need surfaces.
- **Page wiring (`app/bai-viet/[slug]/page.tsx`, US00066):** compute `const toc = extractToc(post.content)`. Pass `<TableOfContents entries={toc} />` as `<ShellLayout leftPanel={…}>`. When `toc.length === 0`, the component returns `null`; the page should also avoid a dead/empty left panel (e.g. render the post full-width or let `<ShellLayout>` collapse) so the body reads normally.
- **Mobile — hidden** (C3). The TOC renders only on desktop. Hide it via CSS
  (`MOBILE_MEDIA_QUERY` / `display:none <768px`) rather than routing into `<Drawer>`. Rationale:
  TOC has low mobile value, the drawer is already filter real estate on listings, and the spec
  allows "hidden". The Vietnamese left-panel labels (`leftPanelTitle="Mục lục"`) are still set
  for the desktop sticky title; mobile users get a clean linear read.

## 7. Edge cases

- **No `h2`/`h3` (F0006 §5):** `extractToc` returns `[]`; component renders `null`; page falls back to full-width body. Covered by AC3.
- **Only `h3`s, or `h3` before any `h2`:** render flat / best-effort nesting; never crash.
- **Duplicate heading text:** rely on the shared slugger's `-1`/`-2` suffixing so anchors stay unique and match US00062 exactly.
- **Code blocks containing `## …`:** avoided by AST parse, not regex.
- **Very long TOC:** `overflow:auto` + `max-height` so the sticky panel scrolls independently.
- **Sticky header overlap:** `scroll-margin-top` on headings.
