## 1. Story summary

Attach baseline SEO metadata to the existing static About page at `/ve-chung-toi/`:

- A unique Vietnamese `<title>` and `<meta name="description">`.
- `alternates.canonical = ${NEXT_PUBLIC_SITE_URL}/ve-chung-toi/` (trailing slash).
- Env-var handling that does **not** crash `next build` when `NEXT_PUBLIC_SITE_URL` is missing, matching the existing project convention.

This story is **SEO-baseline only**. JSON-LD, Open Graph, breadcrumb metadata, and sitemap inclusion are explicitly **deferred to F0009** and must **not** be added here.

## 3. Existing convention to reuse (do not invent a new one)

A scan of every page in `app/` that sets `alternates.canonical` shows two variants in use:

| File | Pattern |
|---|---|
| `app/page.tsx` (homepage) | `` `${process.env.NEXT_PUBLIC_SITE_URL ?? ""}/` `` |
| `app/cong-bo-tiep-thi-lien-ket/page.tsx` | `` `${process.env.NEXT_PUBLIC_SITE_URL ?? ""}/cong-bo-tiep-thi-lien-ket/` `` |
| `app/san-pham/[slug]/page.tsx` | inside a `generateMetadata`, env-var-safe via fallback |
| `app/bai-viet/[slug]/page.tsx` | inside a `generateMetadata`, env-var-safe via fallback |
| `app/chinh-sach-bao-mat/page.tsx` | `` `${process.env.NEXT_PUBLIC_SITE_URL}/chinh-sach-bao-mat/` `` (no fallback) |
| `app/san-pham/page.tsx` | `` `${process.env.NEXT_PUBLIC_SITE_URL}/san-pham/` `` (no fallback) |
| `app/bai-viet/page.tsx` | `` `${process.env.NEXT_PUBLIC_SITE_URL}/bai-viet/` `` (no fallback) |
| `app/danh-muc/[category]/page.tsx` | relative `/danh-muc/${meta.slug}` (no host) |

**Decision:** Adopt the **`?? ""` variant** for US00103. Reasons:

1. The story's Scenario 2 explicitly requires that "a missing `NEXT_PUBLIC_SITE_URL` does not crash the build (matches existing project behavior)". The `?? ""` form is the strictest interpretation of "does not crash" — without it, a missing env var produces a canonical literal containing the string `"undefined"`, which is a soft bug (no crash, but invalid URL). The `?? ""` form degrades to a relative `/ve-chung-toi/` instead, which is recoverable downstream.
2. It is the pattern used by the two most-recent precedents — the homepage (`app/page.tsx`) and the closest sibling policy page (`app/cong-bo-tiep-thi-lien-ket/page.tsx`) — and by the closest F0010 sibling story (F0005).
3. Do **not** retrofit the older no-fallback pages in this story; that is out of scope and unrelated to the About page.

## 5. Exact metadata block to add

```tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: "Về chúng tôi | aff-store",
  description:
    "aff-store là trang đánh giá và gợi ý mua sắm thiết bị gaming, công nghệ tại Việt Nam. Tìm hiểu sứ mệnh, đội ngũ và cách chúng tôi chọn sản phẩm để giới thiệu.",
  alternates: {
    canonical: `${process.env.NEXT_PUBLIC_SITE_URL ?? ""}/ve-chung-toi/`,
  },
};
```

Notes:

- **Title format** mirrors the F0005 sibling pages exactly (`"… | aff-store"`). Do **not** import `SITE_NAME` just for this — the F0005 pages hard-code `"aff-store"` and US00103 should match its cohort, not invent a third variant.
- **Description** is a single Vietnamese sentence (≈160 chars), summarizing the three editorial blocks US00101 ships (mission, who we are, how we pick products). It must be unique across the site; spot-check that no other page already uses this string.
- **Canonical** uses trailing slash (`/ve-chung-toi/`) to match the header nav (`lib/nav-items.ts`) and footer "Giới thiệu" link slugs, and the site's existing route style (all F0005 pages, homepage `/`, etc.).
- **No `metadataBase`** is set here; existing pages don't set one either, so canonical must remain a fully-qualified template literal driven by `NEXT_PUBLIC_SITE_URL`.

## 10. Edge cases & guards

- **Missing `NEXT_PUBLIC_SITE_URL`:** `?? ""` fallback prevents the literal string `"undefined"` from appearing in the canonical and keeps build green. Aligns with the F0010 edge-case note: "Canonical generation must follow the same env-var handling used by other pages; a missing value must not crash the build."
- **Trailing slash:** Must be `/ve-chung-toi/`, not `/ve-chung-toi`. Mismatching the nav-item slug would produce two distinct canonicals if someone links to the non-slash form externally.
- **Title uniqueness:** Spot-check `app/**/page.tsx` and `app/**/[slug]/page.tsx` to confirm no other page already exports `"Về chúng tôi | aff-store"` (none should — verified during planning).
- **Description uniqueness:** Confirm the §5 description string does not collide with the F0005 policy-page descriptions, the homepage, or any blog/product listing description.
- **No `metadataBase` regression:** Do not add a `metadataBase` to this file in the name of "cleaning up" the canonical — that is a site-wide decision out of scope for US00103.
