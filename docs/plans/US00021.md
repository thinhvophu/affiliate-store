# Implementation Plan — US00021: Shopee-style Site Header

**Status:** Approved
**Feature:** F0002 — Site Shell & Theme
**Spec ref:** `docs/specs/F0002.md` § US00021
**Predecessors / dependencies:**

- **US00016** (Done) — `app/globals.css` already defines `--color-primary: #ee4d2d`, breakpoint tokens (`--breakpoint-tablet: 768px`, `--breakpoint-desktop: 1280px`), spacing scale, `--max-width: 1280px`, system font stack.
- **US00023** (sibling, this feature) — adds Shopee surface tokens this story references (`--color-on-primary`, `--color-page-bg`, `--color-surface`, `--color-border-hairline`, `--radius-sm`, `--radius-md`) **and** wires `<Header />` into `app/layout.tsx`. **Hard dependency for visual correctness.** Until US00023 lands, the Header will fall back to the literal `#ffffff` text color via the token's `var(..., #fff)` fallback this plan specifies, so US00021 can be built and merged independently of US00023's merge order, but the two should ship together to look right.

**Related (out of scope here, named so the boundaries are clean):**

- **US00022** — Footer (sibling).
- **US00025** — Mobile drawer behavior (panel slide-in, overlay, focus trap, body scroll lock, Esc handling). US00021 only renders the **trigger button** and dispatches an "open" intent.
- **US00026** (P2) — Sticky-on-scroll header behavior. Out of scope here; the markup this story produces must be compatible with a later sticky wrapper but must not implement it.

## 1. Scope

Strictly:

1. Create `components/Header.tsx` — a **Server Component** that renders the orange Shopee brand band: `<header>` containing a `<nav>` with the site logo, site name, and exactly the four primary nav links in this order — `Trang chủ` (`/`), `Sản phẩm` (`/san-pham`), `Bài viết` (`/bai-viet`), `Về chúng tôi` (`/ve-chung-toi`).
2. Create `components/HeaderNav.tsx` — a small `"use client"` component that reads `usePathname()` from `next/navigation` and renders the four nav links with active-route detection (prefix match, see §3 D2). Each link is a `next/link` `<Link>`.
3. Create `components/HeaderMobileTrigger.tsx` — a small `"use client"` component that renders the hamburger button visible only `<768px`. The trigger fires a documented intent that the **US00025 drawer** subscribes to (see §3 D5 for the contract). US00021 does **not** implement the drawer itself, does **not** open any panel, does **not** trap focus, and does **not** lock body scroll.
4. Create `components/Header.module.css` — scoped styles for the Header (orange band, spacing, focus ring, mobile/desktop toggling of nav vs hamburger, site-name truncation).
5. Verify the Header renders correctly via a temporary local mount on `app/page.tsx` (not committed) — root-layout wiring is owned by **US00023** and is not part of this story.

**Explicitly out of scope:**

- Wiring `<Header />` into `app/layout.tsx` — US00023.
- Footer markup — US00022.
- Drawer panel, overlay, focus trap, body scroll lock — US00025.
- Sticky-on-scroll behavior + shadow — US00026.
- The new Shopee surface tokens themselves (`--color-on-primary`, `--color-page-bg`, `--color-surface`, `--color-border-hairline`, `--radius-sm`, `--radius-md`) — declared by US00023. This plan **references** them via `var(--color-on-primary, #ffffff)` with explicit fallbacks so the Header is visually correct even if US00021 lands first.
- Skip-to-main-content link — US00023 (lives in the root layout).
- Search bar / category quick-links band — out of v1.
- Cart / auth / saved icons — out of project scope.

### 5.3 `components/HeaderNav.tsx`

```tsx
"use client";

import Link from "next/link";
import { usePathname } from "next/navigation";
import { NAV_ITEMS, type NavItem } from "@/lib/nav-items";
import styles from "@/components/Header.module.css";

function isActive(pathname: string, href: string): boolean {
  if (href === "/") return pathname === "/";
  return pathname === href || pathname.startsWith(`${href}/`);
}

export function HeaderNav() {
  const pathname = usePathname() ?? "/";

  return (
    <nav className={styles.nav} aria-label="Điều hướng chính">
      <ul className={styles.navList}>
        {NAV_ITEMS.map((item: NavItem) => {
          const active = isActive(pathname, item.href);
          return (
            <li key={item.href} className={styles.navItem}>
              <Link
                href={item.href}
                className={`${styles.navLink} ${active ? styles.navLinkActive : ""}`.trim()}
                aria-current={active ? "page" : undefined}
              >
                {item.label}
              </Link>
            </li>
          );
        })}
      </ul>
    </nav>
  );
}
```

```tsx
"use client";

import { useState } from "react";

import styles from "@/components/Header.module.css";

/**
 * Mobile-only menu: hamburger trigger + drawer panel containing the nav links.
 * Self-contained — owns its own open/close state. Uses the shared <Drawer>
 * primitive from US00025 (confirmed by the F0002 decisions walkthrough).
 *
 * Hand-off ordering:
 *   - If US00025's <Drawer> is not yet on main when this story is implemented,
 *     ship with the simple inline <div> panel below; swap to <Drawer> in a
 *     one-line follow-up commit once US00025 merges.

export function HeaderMobileMenu() {
  const [open, setOpen] = useState(false);
  const close = () => setOpen(false);

  return (
    <>
      <button
        type="button"
        className={styles.hamburger}
        aria-label="Mở menu"
        aria-haspopup="dialog"
        aria-expanded={open}
        aria-controls="header-mobile-nav"
        onClick={() => setOpen(true)}
      >
        <svg viewBox="0 0 24 24" width="24" height="24" aria-hidden="true" focusable="false">
          <path d="M4 6h16M4 12h16M4 18h16" stroke="currentColor" strokeWidth="2" strokeLinecap="round" />
        </svg>
      </button>

      {/* Once US00025 lands, replace this block with:
          <Drawer open={open} onClose={close} ariaLabel="Điều hướng chính" side="right">
            <nav id="header-mobile-nav" aria-label="Điều hướng chính">…</nav>
          </Drawer>
      */}
      {open && (
        <div className={styles.mobileNavPanel} role="dialog" aria-label="Điều hướng chính" id="header-mobile-nav">
          <button type="button" className={styles.mobileNavClose} aria-label="Đóng menu" onClick={close}>

            <ul className={styles.mobileNavList}>
              {NAV_ITEMS.map((item: NavItem) => (
                <li key={item.href}>
                  <Link href={item.href} onClick={close} className={styles.mobileNavLink}>
                    {item.label}
                  </Link>
                </li>
              ))}
            </ul>
          </nav>
        </div>
      )}
    </>
  );
}
```

## 10. Risks / open items

- **Token availability (US00023 order).** If US00021 ships before US00023, the `var(..., #fallback)` values keep the Header visually correct, but the Header is also **not mounted in the root layout** until US00023 wires it. Per the F0002 decisions walkthrough, six separate PRs ship in dep order — expect US00021 → US00023 with the Header unmounted briefly. That is safe.
- **`<Drawer>` primitive availability (US00025 order).** If US00021 lands before US00025, the inline-panel fallback in §5.4 keeps the mobile menu functional. A one-line follow-up commit swaps to `<Drawer>` once US00025 merges. Tracked in §11 DoD.
- **Drawer state is owned by `HeaderMobileMenu`.** Header is no longer purely a Server Component for the mobile-menu subtree — `HeaderMobileMenu` is `"use client"` and holds its own `useState` for open/close. `aria-expanded` is driven by that state (not statically `"false"`). The static `Header.tsx` shell remains a Server Component.
- **Site name placeholder.** `aff-store` is a working title. If marketing renames before launch, edit `SITE_NAME` in `Header.tsx` — single source.
- **Nav targets 404 before F0004/F0006/F0010.** Same pattern as the Footer's Privacy/Disclosure reserved paths — acceptable interim, no link updates required when those features land.
- **Header-height token.** US00021 uses `var(--header-height, 64px)`; US00026 ships the canonical `:root` declaration. If US00021 lands first, the literal fallback applies.
