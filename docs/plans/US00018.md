# Implementation Plan — US00018: Typed environment variable access

**Status:** Approved
**Feature:** F0001 — Technical Foundation & Bootstrap
**Spec ref:** `docs/specs/F0001.md` § US00018
**Predecessors:** US00011 (Next.js + TS bootstrap), US00012 (canonical folders + `@/*` alias)

---

## 1. Scope

Strictly:

1. Create a single typed module `lib/env.ts` that is the **only** place in the codebase that touches `process.env.*`.
2. Validate `NEXT_PUBLIC_SITE_URL` as a required, well-formed URL — missing or invalid fails the build with a clear, named-variable error message.
3. Expose `NEXT_PUBLIC_GA_MEASUREMENT_ID` as optional — empty/unset is OK (analytics disabled; consumed later by F0007).
4. Ensure validation runs at **build time** (module-level, not lazy/on-demand).
5. Provide a custom ESLint rule (or rule config) to forbid raw `process.env.*` access outside `lib/env.ts` — coordinates with US00017's lint config.

**Explicitly out of scope** (owned by sibling stories in F0001):

- Shared TS types (`Product`, `Post`) -> **US00014**
- Content loader utilities -> **US00015**
- Global styles / design tokens -> **US00016**
- ESLint + Prettier baseline config -> **US00017** (but this story adds one custom rule that plugs into US00017's config)
- MDX pipeline -> **US00013**
- GA4 component / analytics wiring -> **F0007**

## 5. Concrete diff sketch

### 5.1 `lib/env.ts`

```ts
/**
 * Typed environment variable access.
 *
 * This module is the ONLY place in the codebase that reads process.env.*.
 * All other files import { env } from "@/lib/env" instead.
 *
 * Validation runs at module-evaluation time. Because this module is imported
 * by the root layout, any missing/invalid required variable will crash the
 * build with a clear error message.
 */

function requireEnvVar(name: string): string {
  const value = process.env[name];
  if (!value) {
    throw new Error(
      `Environment variable ${name} is required but was not set.\n` +
        `Add it to .env.local or your hosting provider's environment settings.`,
    );
  }
  return value;
}

function requireUrl(name: string): string {
  const value = requireEnvVar(name);
  try {
    new URL(value);
  } catch {
    throw new Error(`Environment variable ${name} must be a valid URL.\n` + `Received: "${value}"`);
  }
  return value;
}

function optionalEnvVar(name: string): string | undefined {
  const value = process.env[name];
  return value || undefined;
}

// --- Validation runs HERE, at module evaluation time ---

export const env = {
  /** Base URL for canonical links and sitemap. Required; must be a valid URL. */
  NEXT_PUBLIC_SITE_URL: requireUrl("NEXT_PUBLIC_SITE_URL"),

  /**
   * Google Analytics 4 measurement ID (e.g. "G-XXXXXXXXXX").
   * Optional — when empty/unset, analytics is disabled (see F0007).
   */
  NEXT_PUBLIC_GA_MEASUREMENT_ID: optionalEnvVar("NEXT_PUBLIC_GA_MEASUREMENT_ID"),
} as const;

export type Env = typeof env;
```

### 5.4 ESLint rule snippet (for US00017 to integrate)

This is the `no-restricted-syntax` configuration that enforces "no raw `process.env` outside `lib/env.ts`". It must be added to the ESLint config that US00017 creates.

```js
// In the ESLint config's rules section:
{
  "no-restricted-syntax": [
    "error",
    {
      "selector": "MemberExpression[object.object.name='process'][object.property.name='env']",
      "message": "Do not access process.env directly. Import { env } from '@/lib/env' instead."
    }
  ]
}

// With an override to exempt lib/env.ts:
{
  "overrides": [
    {
      "files": ["lib/env.ts"],
      "rules": {
        "no-restricted-syntax": "off"
      }
    }
  ]
}
```

This snippet is **documented here** for the US00017 implementer to pick up. It is not committed as a standalone file in this story because there is no ESLint config to plug it into yet.

## 10. Risks / open items

- **US00017 dependency for lint enforcement.** The ESLint `no-restricted-syntax` rule (AC Scenario 4) cannot be enforced automatically until US00017 creates the ESLint config. This story verifies the constraint manually (grep). The rule snippet in Section 5.4 must be picked up by the US00017 implementer. If US00017 lands first, this story's implementer should integrate the rule directly.
- **`NEXT_PUBLIC_*` vars are inlined by Next.js at build time.** This is standard behavior and actually reinforces our build-time validation: the values are baked into the static output, so runtime access is not needed. No special handling required.
- **`as const` assertion on the `env` object.** This narrows the types but means `env.NEXT_PUBLIC_GA_MEASUREMENT_ID` is typed as `string | undefined`, not `string`. Consumers must handle the `undefined` case — which is correct (analytics may be disabled).
- **`.env.local` is git-ignored by default** (Next.js convention + `.gitignore` from US00011). `.env.example` serves as documentation for required vars. No secrets are committed.
- **No Zod dependency.** The validation is intentionally hand-rolled (two variables, simple rules). If F0001 scope grows to more env vars, reconsider adding a schema library.
- **`lib/.gitkeep` removal.** If US00015 (content loaders) lands before US00018 and has already removed `.gitkeep`, this story should skip the deletion step (it will be a no-op).
