## 3. Key decisions

### 3.1 Canonical URL — trailing slash, absolute string

The spec mandates the canonical be exactly `${NEXT_PUBLIC_SITE_URL}/chinh-sach-bao-mat/` (trailing
slash). The repo has **two** divergent canonical patterns:

- `app/san-pham/page.tsx` → absolute string `` `${process.env.NEXT_PUBLIC_SITE_URL}/san-pham/` `` (trailing slash).
- `app/danh-muc/[category]/page.tsx` → relative `` `/danh-muc/${slug}` `` (no trailing slash).

`next.config.ts` sets **no `trailingSlash`** (default `false`) and **no `metadataBase`**. A relative
canonical would (a) error without `metadataBase` or (b) be normalized to strip the trailing slash.
**Decision:** follow the `san-pham` pattern — emit the absolute string
`` `${process.env.NEXT_PUBLIC_SITE_URL}/chinh-sach-bao-mat/` `` so the trailing slash is preserved
verbatim (per `generate-metadata.md`: "If a `metadata` field provides an absolute URL,
`metadataBase` will be ignored"). This satisfies Scenario 4 exactly.

### 3.2 Footer "broken link" — no edit needed (verify only)

The task prompt says to "fix the currently-broken footer link." Inspection of
`components/Footer.tsx` shows the link target `/chinh-sach-bao-mat/` is **already present and
correct** (added in US00022). It is "broken" only because the route 404s today. **Creating the
route file is the fix** — no Footer change is required. Scenario 2 is satisfied by route existence.
(Open question — see §7.)

### 3.3 Static, no dynamic segment

Leaf static route. No `generateStaticParams`. Per spec edge case, `/chinh-sach-bao-mat/anything`
404s naturally because there is no dynamic `[slug]` child. Page is SSG by default (Server Component,
no dynamic APIs, no `dynamic` export).

### 3.4 Page-scoped prose styling

No shared "prose/legal" layout component exists. Add a co-located
`app/chinh-sach-bao-mat/chinh-sach-bao-mat.module.css` for a constrained reading-width container
(`max-width`), heading hierarchy spacing, and list/paragraph rhythm, using existing design tokens
from `globals.css` (`--color-primary`, surface tokens). System font stack only (no external fonts).
**Decision (resolved):** use a plain centered `<section>`/`<article>` prose wrapper — **not**
`ShellLayout`. Legal pages have no left-panel content (no filters/TOC), so the two-column shell
would render an empty column. A constrained max-width reading column is the right layout for
long-form legal text.

## 5. Page content outline (real Vietnamese copy)

`<h1>`: **"Chính sách bảo mật"**. Lead paragraph: effective date + plain-language scope statement.
Then semantic `<section>` blocks, each with an `<h2>`:

1. **Thông tin chúng tôi thu thập** — site collects **no accounts and no PII**; the only data is
   anonymized GA4 analytics, and **only when** `NEXT_PUBLIC_GA_MEASUREMENT_ID` is configured (per
   `CLAUDE.md` env rules); GA never loads in development.
2. **Cookie và công nghệ theo dõi** — cookies set by Google Analytics / AdSense when enabled; the
   site itself sets no login/cart cookies (no auth, no cart per project scope).
3. **Dịch vụ bên thứ ba** — Shopee affiliate program (clicks open `shope.ee`/`shopee.vn` links in a
   new tab; the `affiliate_click` GA4 event records product name/category/destination URL), Google
   Analytics, and Google AdSense; link out to their respective privacy policies.
4. **Liên kết tiếp thị liên kết** — short reference to the affiliate relationship, pointing readers
   to `/cong-bo-tiep-thi-lien-ket/` for the full disclosure (no duplication of the F0005 constant
   text).
5. **Quyền của bạn & liên hệ** — contact point: email `ttln1201@gmail.com` (same address used in
   `Footer.tsx` `CONTACT_EMAIL`).
6. **Thay đổi chính sách** — note that the policy may be updated; check back for changes.

All copy is good-faith Vietnamese prose (not lorem ipsum, not legal-reviewed — legal sign-off is
explicitly out of scope per F0005 §2).
