## 1. Story summary

On the homepage (`/`), render a **"BĂ i viáşżt má»›i nháşĄt"** section that shows the **4 newest blog posts**

JS. Dates are rendered by `<PostCard>` via `formatPostDate()` (single chokepoint preserved).

## 3. Component design â€” `<LatestPosts />`

### 3.1 Files to create

- `components/LatestPosts.tsx` (Server Component)
- `components/LatestPosts.module.css` (scoped grid + heading + footer link styles)

### 3.2 Props & contract

```ts

export const LATEST_POSTS_COUNT = 4; // named constant (per F0008 Â§6: caps are explicit)

  posts: Post[]; // expects newest-first; component slices to LATEST_POSTS_COUNT defensively

```

- Receives an array of posts (caller passes `getAllPosts()` â€” homepage already needs the loader for
  this strip; no second disk read inside the component).
- Component **slices to `LATEST_POSTS_COUNT`** internally so the cap is enforced regardless of caller.
- **No props for heading text or count** â€” this is a single-purpose section. If F0008 reuses the same
  shape elsewhere later, generalize then.

### 3.3 Render shape

```tsx
import Link from "next/link";
import { PostCard } from "@/components/PostCard";
import type { Post } from "@/types";
import styles from "./LatestPosts.module.css";

export const LATEST_POSTS_COUNT = 4;

export interface LatestPostsProps {
  posts: Post[];
}

export function LatestPosts({ posts }: LatestPostsProps) {
  const latest = posts.slice(0, LATEST_POSTS_COUNT);

  // Scenario 4: omit entirely when no posts exist.
  if (latest.length === 0) return null;

  return (
    <section className={styles.section} aria-labelledby="latest-posts-heading">
      <header className={styles.header}>
        <h2 id="latest-posts-heading" className={styles.heading}>
          BĂ i viáşżt má»›i nháşĄt
        </h2>
        <Link href="/bai-viet/" className={styles.seeAll}>
          Xem táşĄt cáşŁ
        </Link>
      </header>
      <div className={styles.grid} data-count={latest.length}>
        {latest.map((post) => (
          <PostCard key={post.slug} post={post} />
        ))}
      </div>
    </section>
  );
}
```

**Heading level:** `<h2>` â€” the homepage already has a single `<h1>` in the hero (US00081); F0008
sections must use `<h2>`.

**Accessibility:** `aria-labelledby` ties the `<section>` landmark to its heading; the "Xem táşĄt cáşŁ"
link is a real `<Link>` (crawlable). No `aria-label` on the `<Link>` â€” visible text is sufficient
and Vietnamese.

**No client JS:** Server Component, no `"use client"`. No event handlers, no state.

### 3.4 Responsive grid

Mirror `PostListingClient.module.css` (the canonical post-grid breakpoints) **adapted to a max of
4 cards**:

| Viewport | Columns | Cards shown if N=4 |
| --- | --- | --- |
| Mobile (default, <768px) | 2 | 2Ă—2 |
| Tablet (â‰Ą768px, <1280px) | 2 | 2Ă—2 |
| Desktop (â‰Ą1280px) | 4 | 1Ă—4 |

Rationale for **4-up at desktop** (vs. 3-up like `PostListingClient`): the cap is 4 and the strip
should read as a single horizontal row at desktop to keep the homepage scannable. The tablet
breakpoint stays at 2 cols so cards remain comfortably wide between 768â€“1280px.

When `N < 4`, the grid still uses `repeat(N, 1fr)` style (via `data-count` attr) at desktop only
so the row doesn't stretch sparse cards across the full width:
