## 1. Scope

This story has **two distinct surfaces**:

### 1A. Vercel dashboard configuration (no code change)

Set environment variables in the Vercel project's **Settings → Environment Variables** UI:

1. `NEXT_PUBLIC_SITE_URL` — set in **Production** to the canonical absolute URL (the Vercel-assigned `*.vercel.app` URL until launch swaps in the custom domain). Must be a valid URL or the build fails per US00018.
2. `NEXT_PUBLIC_SITE_URL` — set in **Preview** to a Preview-appropriate value (e.g. the same Production URL, or `https://aff-store-git-preview.vercel.app`-style placeholder). The build still needs a valid URL — empty fails per US00018.
3. `NEXT_PUBLIC_GA_MEASUREMENT_ID` — defined as a key in **Production** and **Preview**, value left **empty** (per F0007 — analytics not yet wired). Defining the key (even empty) keeps the env-var inventory explicit and aligned with `lib/env.ts`'s `optionalEnvVar` contract.

### 1B. Repo change — `.env.example` (code change)

Confirm `.env.example` exists at the repo root and documents the same two keys. **It already exists today** (committed in US00018 — verified in §2 below). Re-validate that:

- It is checked into git.
- It documents both `NEXT_PUBLIC_SITE_URL` (required, with a placeholder URL) and `NEXT_PUBLIC_GA_MEASUREMENT_ID` (optional, empty by default).
- No real secrets / production values are present.
- `.gitignore` continues to ignore `.env*.local` so developer-local overrides never get committed.

**Explicitly out of scope** (owned by sibling stories in F0011):

- Linking the Vercel project / setting Production branch → **US00111**
- Scheduled-rebuild workflow + `VERCEL_DEPLOY_HOOK_URL` GitHub secret → **US00113**
- PR validation CI (`.github/workflows/ci.yml`) + branch protection → **US00114**
- Speed Insights enablement → **US00115**
- GA4 component / analytics wiring (the actual consumption of `NEXT_PUBLIC_GA_MEASUREMENT_ID`) → **F0007**
- Custom domain / DNS → launch prep (out of F0011 entirely)

## 3. Approved Decisions

| #   | Decision                                          | Choice                                                                                                                                        | Rationale                                                                                                                                                                                                                                  |
| --- | ------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| D1  | `NEXT_PUBLIC_SITE_URL` Production value           | **The Vercel-assigned canonical URL** (e.g. `https://aff-store.vercel.app`) until custom domain lands.                                        | F0011 ships before custom-domain DNS. | D2  | `NEXT_PUBLIC_SITE_URL` Preview value              | **The same Production URL** (or a Preview-specific placeholder if the operator prefers).                                                      | Preview deploys also import `lib/env.ts`, which requires a valid URL. | D3  | `NEXT_PUBLIC_GA_MEASUREMENT_ID` value             | **Defined as a key, value empty** in both Production and Preview.                                                                             | `lib/env.ts` treats `""` as `undefined` (`value \|\| undefined`). Skip Vercel's "Development" env.                                                                                    | Local dev uses `.env.local`. The Vercel "Development" env only matters for `vercel dev`, which the project does not use.                                                                                                                   |
| D5  | "Sensitive" flag in Vercel UI                     | **Off** for both vars.                                                                                                                        | They are `NEXT_PUBLIC_*` — by definition exposed to the client bundle. Marking them sensitive offers no security benefit and complicates retrieval.                                                                                        |
| D6  | `.env.example` action                             | **Verify, do not modify.**                                                                                                                    | File already exists (US00018) and documents both keys correctly. Touching it would create a no-op diff. AC §5 is satisfied by the existing file.                                                                                           |
| D7  | Capture configuration evidence                    | **Screenshots of Vercel Env Vars UI** (Production + Preview tabs) attached to the story for audit; not committed to the repo.                 | F0011 AC §1–3 are dashboard-state assertions. Screenshots are the only durable evidence; the repo intentionally does not store cloud config.                                                                                               |
| D8  | Trigger a verification redeploy after configuring | **Yes** — after setting the vars, push a no-op commit (or re-run latest deploy from Vercel UI) to confirm Production build succeeds.          | AC §1 / §3 are observable only via a successful build that consumes the vars. AC §4 (missing-var fail) is a tabletop check (do **not** actually delete the var on Production); see §7.                                                     |
| D9  | Scope of branch / PR                              | **No PR needed for §1A.** §1B is verification-only and produces no diff. ## 5. Step-by-step execution order

1. **Confirm prerequisites.** Verify US00111 is complete: open the Vercel project; confirm it is linked to the GitHub repo and Production branch is `main`. If not, stop — this story is blocked on US00111.
2. **Open Vercel → Project → Settings → Environment Variables.**
3. **Add `NEXT_PUBLIC_SITE_URL`:**
   - Key: `NEXT_PUBLIC_SITE_URL`
   - Value: the canonical Production URL (D1) — e.g. `https://aff-store.vercel.app`
   - Environments: tick **Production** and **Preview**
   - Save
4. **Add `NEXT_PUBLIC_GA_MEASUREMENT_ID`:**
   - Key: `NEXT_PUBLIC_GA_MEASUREMENT_ID`
   - Value: leave **empty**
   - Environments: tick **Production** and **Preview**
   - Save
5. **Capture screenshots** of the Production and Preview tabs of the Env Vars UI for audit (D7).
6. **Verify locally** that `.env.example` is correct and tracked:
   - `git ls-files .env.example` returns the path.
   - File still documents both keys with the comments shown in §2.
   - No `.env.production` (or similar real-value file) is in the working tree or staged.
7. **Trigger a Production redeploy:**
   - In Vercel → Deployments, click "Redeploy" on the latest `main` deployment, **without** "Use existing build cache".
   - Confirm the build picks up the new env vars and succeeds.
8. **Verify the Preview path.** On the next PR (or a throwaway branch push), confirm Vercel creates a Preview deploy and the build succeeds with the Preview-scoped env vars.
9. **Run AC verification round** (Section 7).
10. **Close the story.** No commit / push needed unless §1B produced a diff. If a diff exists (rare), branch `feat/US00112`, commit `feat(US00112): clarify env var docs in .env.example`, open PR.

## 9. Risks / open items

- **Custom domain timing.** `NEXT_PUBLIC_SITE_URL` will need to be updated when DNS lands. Note in launch-prep checklist; not actionable now.
- **Vercel "Development" environment.** Out of scope (D4). If a future story needs `vercel dev`, the keys will need to be added to Development too.
- **AC §4 verification is destructive.** Do **not** run the missing-var test on Production — only on Preview / a throwaway branch. See §7.
- **`NEXT_PUBLIC_*` vars are inlined at build time** (Next.js convention). Changing the value in Vercel without a redeploy will not affect the live site. Always trigger a redeploy after editing.
- **Empty `NEXT_PUBLIC_GA_MEASUREMENT_ID` is intentional.** F0007 will flip the value when GA4 wiring lands; this story explicitly does not.
- **US00111 dependency.** If US00111 is not yet merged, the Vercel project doesn't exist for this story to configure. Block on US00111 before starting.
- **Operator access to Vercel.** This story assumes the implementer has Vercel project access. If not, the story is a coordination task, not a self-serve one.
