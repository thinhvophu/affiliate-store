# US00064 — Blog post listing page `/bai-viet/`

**Feature:** F0006 (Editorial Content)
**Status:** Approved
**Priority:** P0 (blog discovery surface)

---

## 1. Story

**As a** site visitor,
**I want to** browse all blog posts newest-first at `/bai-viet/`,
**So that** I can discover buying guides and reviews.

Deliver a statically pre-rendered listing of posts ordered by `publishedAt` descending, each
item showing cover image, linked title, 2–3 sentence summary, formatted date (`formatPostDate`,
US00061), and category tag. 12 posts per page via the shared `<Pagination>` with crawlable
`?page=N` links (`basePath=/bai-viet/`), a friendly Vietnamese empty state, and baseline SEO
metadata (unique Vietnamese title/description, canonical `${NEXT_PUBLIC_SITE_URL}/bai-viet/`).

This story builds **only** the listing. The left-panel category/tag filter (US00065) and the
read-time estimate (US00069) attach to this same page in later stories; design the listing item
and page shell so they slot in without rework, but **do not build them here**.

## 6. Implementation detail

### 6.1 `app/bai-viet/page.tsx`
```tsx
import { Suspense } from "react";
import type { Metadata } from "next";
import { PostListingClient } from "@/components/PostListingClient";
import { getAllPosts } from "@/lib/posts";
import styles from "./page.module.css";

export const metadata: Metadata = {
  title: "Bài viết & hướng dẫn mua sắm | TechShop",
  description:
    "Đánh giá, hướng dẫn mua sắm và mẹo chọn thiết bị gaming, công nghệ — cập nhật mới nhất.",
  alternates: { canonical: `${process.env.NEXT_PUBLIC_SITE_URL}/bai-viet/` },
};

export default function BaiVietPage() {
  const posts = getAllPosts(); // already sorted publishedAt desc
  return (
    <>
      <h1 className={styles.pageHeading}>Bài viết</h1>
      <Suspense fallback={<div className={styles.gridSkeleton} />}>
        <PostListingClient posts={posts} />
      </Suspense>
    </>
  );
}
```
- Static by default (no `dynamic`/`revalidate`); pre-rendered at build.
- Canonical uses the **bare** path with trailing slash — never reflects `?page`/filter params
  (matches the `san-pham` D15 comment).
- `<Suspense>` required because the child uses `useSearchParams()` (App Router CSR-bailout rule —
  confirm against `node_modules/next/dist/docs/` for the current Next version during implementation).

### 6.2 `components/PostListingClient.tsx` (`"use client"`)
Mirror `ProductListingClient` exactly:
- `const PAGE_SIZE = 12;` `const BASE_PATH = "/bai-viet/";`
- Read `?page` via `useSearchParams`; default 1.
- `totalPages = Math.max(1, Math.ceil(posts.length / PAGE_SIZE))`.
- `posts.length === 0` → `<p className={styles.emptyState}>Chưa có bài viết nào.</p>`
- Out-of-range / non-integer page → `<p className={styles.emptyState}>Trang không tồn tại.</p>`
- Slice, map to `<PostCard post={p} />`, then `<Pagination currentPage totalPages basePath />`.

### 6.3 `components/PostCard.tsx` (Server Component)
- Props: `{ post: Post }`.
- Renders a `next/link` to `/bai-viet/${post.slug}/` wrapping:
  - cover image frame via `next/image` (`fill` + responsive `sizes` like ProductCard); guard a
    missing `coverImage` gracefully (omit the frame — see Edge Cases). `getAllPosts` already
    requires `coverImage` as a non-empty string, so this is belt-and-braces.
  - `<h2>` linked title (the page `<h1>` is "Bài viết"; each item is an `<h2>`).
  - 2–3-sentence `summary` with a CSS line-clamp.
  - a **meta row**: `formatPostDate(post.publishedAt)` + category tag
    (`getCategoryLabels()[post.category] ?? post.category`).
- **Extensibility seam (no build now):** the meta row is a single flex container with `date` and
  `category` children. US00069 read-time slots in as a sibling in that row; the tag rendering uses
  the same `getCategoryLabels` lookup US00065's filter will reuse. Keep `PostCard` filter-agnostic
  (it receives a `Post`, knows nothing about active filters).

### 6.4 CSS
- `PostListingClient.module.css`: 2-col mobile / 3-col ≥768px / 3-col ≥1280px grid (blog cards are
  wider than product cards; 3 desktop cols read better — confirm in browser, adjust to 4 if cramped),
  `.emptyState` copied from `ProductListingClient.module.css`.
- `PostCard.module.css`: card surface, 16:9 image frame (`object-fit: cover`), title, clamped
  summary, meta row. Use design tokens (`--color-*`, `--spacing-*`).

## 7. Designing for US00065 (filter) without building it

The `san-pham` precedent wraps its grid in `<ShellLayout leftPanel={…}>` + a mobile trigger.
For US00064 (no filter yet) the page renders the grid directly under `<h1>`. To make US00065 a
clean add, **do not** hard-code assumptions that block a left panel:

- Keep `PostListingClient` accepting the **full** `posts` array and doing pagination only. US00065
  will pre-filter `posts` (via a `lib/filters.ts`-style URL helper) before passing them in, or wrap
  this client with a filtered list — either way `PostListingClient` stays a pure paginator.
- Keep `PostCard` filter-agnostic.
- When US00065 lands it will introduce `<ShellLayout leftPanel={<PostFilters/>}>` around this page's
  body and a mobile `<Drawer>` trigger — the same shape as `san-pham`. Nothing in this story should
  prevent that wrap.

This avoids a `ShellLayout` import now (YAGNI) while leaving the obvious slot open.
