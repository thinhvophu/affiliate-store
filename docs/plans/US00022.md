# Implementation Plan — US00022: Multi-column Site Footer

**Status:** Approved
**Feature:** F0002 — Site Shell & Theme
**Spec ref:** `docs/specs/F0002.md` § US00022
**Predecessors / siblings:**

- **Soft dependency on US00023** (root layout + Shopee surface tokens). US00023 introduces `--color-border-hairline`, `--color-surface`, `--color-on-primary`, `--radius-sm`, `--radius-md`, and is the story that actually mounts `<Footer />` inside `app/layout.tsx`. This story (US00022) **ships only the `<Footer />` component file**; US00023 wires it in.
  - If US00022 lands first: the component compiles, but the tokens it references won't yet resolve on `:root`. That's acceptable as long as US00023 lands in the same feature batch — F0002 PRs ship together.
  - Recommended ordering for the implementer: land US00023 first (tokens + wiring), then US00021 (Header) and US00022 (Footer) plug into it. If parallel work is needed, fall back to defining tokens locally on the `<footer>` element as a temporary scoped override (called out in §10 Risks).
- **Sibling US00021** (Header) — independent component, no shared code.
- **F0005** ships `/chinh-sach-bao-mat/` and `/cong-bo-tiep-thi-lien-ket/`. Footer links to those canonical Vietnamese slugs now; the destinations 404 until F0005 ships. This is acceptable per the spec.

## 1. Scope

Strictly:

1. Create a new Server Component `components/Footer.tsx` that renders the site-wide Footer per the F0002 spec.
2. Create a new pure-data module `lib/disclosures.ts` exporting the canonical Vietnamese affiliate-disclosure string as a constant (`AFFILIATE_DISCLOSURE_VI`) so blog posts (F0006) and the Affiliate Disclosure page (F0005) can re-import it without duplicating the copy.
3. Implement the multi-column layout with the four section titles **`Về chúng tôi`**, **`Chính sách`**, **`Liên hệ`**, **`Theo dõi`**, hairline column separators (`var(--color-border-hairline)`), a thin orange top-accent line (`var(--color-primary)`), and the affiliate-disclosure block.
4. Link `/chinh-sach-bao-mat/` (Chính sách bảo mật) and `/cong-bo-tiep-thi-lien-ket/` (Công bố tiếp thị liên kết) inside the `Chính sách` column. These are canonical slugs; F0005 must honor them.
5. Provide at least one contact channel in the `Liên hệ` column (single email link is sufficient; placeholder address documented in §3 D6).
6. Render copyright using `new Date().getUTCFullYear()` evaluated at render time. Since the project is SSG-only, render time **is** build time, so the year is baked into the static HTML.
7. Mobile-stack layout below 768px with all interactive elements meeting a **44×44 px** minimum touch target.
8. Co-locate component styles in `components/Footer.module.css` (CSS Modules) — first new use of CSS Modules in this codebase; convention is documented for re-use.

**Explicitly out of scope** (owned by sibling stories or downstream features):

- Mounting `<Footer />` in `app/layout.tsx` → **US00023**.
- Defining `--color-border-hairline`, `--color-surface`, `--color-on-primary`, `--radius-sm`, `--radius-md` in `app/globals.css` → **US00023**.
- Header → **US00021**.
- Skip-to-main-content link → **US00023**.
- Two-column shell layout component → **US00024**.
- Mobile drawer → **US00025**.
- Sticky header → **US00026**.
- Real destination pages at `/chinh-sach-bao-mat/` and `/cong-bo-tiep-thi-lien-ket/` → **F0005**.
- Real `mailto:` / social handles — placeholder values are fine; the launch checklist (F0011) will replace them. The plan documents the contact-info constant so updating it is a one-line change.
- Cookie-consent banner, GA event wiring, dark mode → all out of v1 scope.

### 5.2 `components/Footer.tsx`

```tsx
import Link from "next/link";
import { AFFILIATE_DISCLOSURE_VI } from "@/lib/disclosures";
import styles from "./Footer.module.css";

const CONTACT_EMAIL = "ttln1201@gmail.com";

export function Footer() {
  const year = new Date().getUTCFullYear(); // SSG: evaluated once at build time, in UTC.

  return (
    <footer className={styles.footer}>
      <div className={styles.topAccent} aria-hidden="true" />

      <div className={styles.container}>
        <section className={styles.column} aria-labelledby="footer-col-about">
          <h2 id="footer-col-about" className={styles.columnTitle}>
            Về chúng tôi
          </h2>
          <ul className={styles.columnList}>
            <li>
              <Link href="/ve-chung-toi/" className={styles.columnLink}>
                Giới thiệu
              </Link>
            </li>
          </ul>
        </section>

        <section className={styles.column} aria-labelledby="footer-col-policy">
          <h2 id="footer-col-policy" className={styles.columnTitle}>
            Chính sách
          </h2>
          <ul className={styles.columnList}>
            <li>
              <Link href="/chinh-sach-bao-mat/" className={styles.columnLink}>
                Chính sách bảo mật
              </Link>
            </li>
            <li>
              <Link href="/cong-bo-tiep-thi-lien-ket/" className={styles.columnLink}>
                Công bố tiếp thị liên kết
              </Link>
            </li>
          </ul>
        </section>

        <section className={styles.column} aria-labelledby="footer-col-contact">
          <h2 id="footer-col-contact" className={styles.columnTitle}>
            Liên hệ
          </h2>
          <ul className={styles.columnList}>
            <li>
              <a href={`mailto:${CONTACT_EMAIL}`} className={styles.columnLink}>
                {CONTACT_EMAIL}
              </a>
            </li>
          </ul>
        </section>

        <section className={styles.column} aria-labelledby="footer-col-follow">
          <h2 id="footer-col-follow" className={styles.columnTitle}>
            Theo dõi
          </h2>
          <p className={styles.columnPlaceholder}>Sắp ra mắt</p>
        </section>
      </div>

      <div className={styles.meta}>
        <small className={styles.disclosure}>{AFFILIATE_DISCLOSURE_VI}</small>
        <p className={styles.copyright}>
          © {year} aff-store. Mọi quyền được bảo lưu.
        </p>
      </div>
    </footer>
  );
}
```

## 10. Risks / open items

- **Token availability if US00022 lands before US00023.** The CSS Module references `--color-surface`, `--color-border-hairline`, `--color-on-primary` which only get defined on `:root` by US00023. Mitigated with fallbacks: `var(--color-surface, #ffffff)` and `var(--color-border-hairline, #e5e5e5)` in §5.3. Visuals will be correct either way. Once US00023 lands, the fallbacks become dead code — leave them as defensive defaults rather than a follow-up cleanup.
- **`<Footer />` not yet rendered.** Until US00023 wires it into `app/layout.tsx`, no route actually displays the Footer. The temporary harness in §6 step 8 makes the component verifiable in isolation. AC Scenarios 1, 2, 3, 7 cannot be verified on a real page until US00023 lands — they're partially verifiable via the harness; the implementer should note this in the PR description.
- **CSS Modules convention is new.** This is the first `*.module.css` in the project. The convention change MUST be reflected in CLAUDE.md / README.md "Module conventions" (per §9). If the user prefers a different styling approach (Tailwind, plain CSS in `globals.css`, etc.), surface it during plan review — this is the decision point.
- **Contact / social column trade-offs.** Per the F0002 decisions walkthrough: contact is `ttln1201@gmail.com` (the site owner's real address — ship-ready, no TODO). `Theo dõi` column renders a single `Sắp ra mắt` placeholder paragraph instead of social links — a future PR can add Facebook/Zalo/TikTok rows without changing markup shape.
- **Policy links 404 until F0005 ships.** Acknowledged by the spec (§5 edge cases). Acceptable. Risk only materializes if F0005 chooses different slugs — then the Footer needs updating in the same PR (spec §6 explicitly notes this).
- **Year boundary edge case.** `getUTCFullYear()` uses UTC, so a build at 23:00 local on Dec 31 in Vietnam (UTC+7) still gets the following UTC year — matches the spec's intent. The site is re-built weekly (Mon 02:00 UTC via the scheduled-deploy workflow), so within ~6 days of New Year the year flips automatically.
- **44×44 enlarged links on desktop.** D8 applies `min-height: 44px` site-wide inside the Footer. On desktop this creates visually loose row spacing — intentional for consistency, and Shopee's actual footer is similarly generous. If the design team objects post-review, switch to a `@media (max-width: 767px)` scope for the touch-target rule.
- **CSS logical properties (`border-inline-start`, `padding-block`) browser support.** Universally supported in all evergreen browsers since 2020. No IE concern (project targets modern browsers only).
- **No drawer / no JS interactivity.** Footer is pure markup — zero client-side state, zero hydration. Stays in the Server Component graph. Confirmed safe per Next.js docs (`05-server-and-client-components.md`).
