## 1. Architecture Decision: SSG + Client-side Pagination

**The core challenge:** Next.js App Router SSG and `?page=N` query params.

If the Server Component reads `searchParams`, the route becomes dynamic (server-rendered per request). The spec explicitly forbids `dynamic = "force-dynamic"`.

**Chosen approach: Pass all products from a static Server Component to a client component. The client component reads `?page` via `useSearchParams()` and paginates in memory.**

- The server page calls `getAllProducts()` at build time, pre-renders page 1 cards as the Suspense fallback (static HTML for SEO + instant paint)
- `<ProductListingClient>` (`"use client"`) wraps inside `<Suspense>` — required by Next.js to keep the page statically generated
- The fallback (page 1 grid) is visible in the pre-rendered HTML — Googlebot sees real products
- After hydration, `ProductListingClient` shows the correct page (reads `useSearchParams()`)
- Pagination links are rendered as real `<a href="?page=N">` inside the client component (crawlable)
- Out-of-range `?page=N` is handled client-side: renders a soft-404 message within the F0002 shell (client components cannot call server-only `notFound()`)

## 2. Approved Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| D1 | Pagination state | Client-side (`useSearchParams()`) | Spec: "the filter UI... receives the full product set from a Server Component parent (already in the HTML), reads the URL query params on mount." Same pattern applies to pagination. |
| D2 | Suspense fallback | Static page-1 product grid (Server Component JSX) | Pre-rendered HTML has real products (good SEO), seamless for page-1 users. Page 2+ users see a brief switch from page 1 → correct page after hydration — acceptable. |
| D3 | Products per page | **24** (spec Scenario 2) | 6 rows × 4 cols desktop / 8 rows × 3 cols tablet / 12 rows × 2 cols mobile |
| D4 | Default sort | Inherit existing `getAllProducts()` sort: `publishedAt` desc, `slug` tiebreaker | Already implemented in `lib/products.ts` line 108. Satisfies Scenario 9 (stable deterministic order). |
| D5 | Out-of-range page | Render soft-404 UI within the F0002 shell | Client components cannot call Next.js's `notFound()`. Spec says "F0002 chrome still wraps it; only the main content is the 404 surface" — met by a "Trang không tồn tại" message in the grid area. |
| D6 | Canonical meta | Static `alternates.canonical: "/san-pham/"` in `export const metadata` | With SSG, per-page canonical requires reading `searchParams` on the server (makes page dynamic). Ship the bare-path canonical now; F0009 hardens pages 2+. Per spec: "coordinate the exact API with F0009 when it lands, but ship the canonical attribute now." |
| D7 | `leftPanel` for US00043 | Pass `null` — empty slot | US00044 fills this. `null` renders an empty left-panel div on desktop (empty white card); acceptable for US00043 verification. |
| D8 | `<Pagination>` component | No `"use client"` directive — pure JSX from props; works in both server and client contexts | Reused by US00045 (category pages). Props: `{ currentPage, totalPages, basePath }`. Page-1 link = bare path (`basePath`); page N link = `${basePath}?page=${N}`. |
| D9 | Pagination links | `<Link>` from `next/link` | Renders as `<a>` (crawlable) + Next.js prefetching + SPA-like navigation (no full reload between pages). |
| D10 | Grid CSS location | `components/ProductListingClient.module.css` | Co-located with the component that renders the grid. Shared with the Suspense fallback via a `.grid` class. |
| D11 | `<h1>` placement | Server Component, above the Suspense boundary | Always present in the static HTML — not gated by the client Suspense. |
| D12 | `images.length` validation | Add to `validateProduct()` in `lib/products.ts` | F0004 §5: "must fail the build with a clear error naming the offending slug." Deferred from US00042 (R3), must be in place before this route ships since it renders product images. |
| D13 | Initial product fixtures | Add 25+ permanent product JSON fixtures to `content/products/` | Browser verification requires real data; pagination testing (Scenario 2) requires >24 products. Product fixtures are content, not test artefacts — they stay in the PR. |

```tsx
import { Suspense } from "react";
import { getAllProducts } from "@/lib/products";
import { ShellLayout } from "@/components/ShellLayout";
import { ProductCard } from "@/components/ProductCard";
import { ProductListingClient } from "@/components/ProductListingClient";
import styles from "@/components/ProductListingClient.module.css";
import type { Metadata } from "next";

const PAGE_SIZE = 24;

export const metadata: Metadata = {
  title: "Tất cả sản phẩm | TechShop",
  description: "Khám phá toàn bộ sản phẩm gaming và công nghệ với giá tốt nhất.",
  alternates: {
    canonical: `${process.env.NEXT_PUBLIC_SITE_URL}/san-pham/`,
  },
};

export default function ProductListingPage() {
  const products = getAllProducts();
  const page1 = products.slice(0, PAGE_SIZE);

  return (
    <ShellLayout leftPanel={null}>
      <h1 className={styles.pageHeading}>Tất cả sản phẩm</h1>
      <Suspense
        fallback={
          <div className={styles.grid}>
            {page1.map((p) => (
              <ProductCard key={p.slug} product={p} />
            ))}
          </div>
        }
      >
        <ProductListingClient products={products} />
      </Suspense>
    </ShellLayout>
  );
}
```

### 4.3 `components/ProductListingClient.tsx` (new)

```tsx
"use client";

import { useSearchParams } from "next/navigation";
import { ProductCard } from "@/components/ProductCard";
import { Pagination } from "@/components/Pagination";
import type { Product } from "@/types";
import styles from "./ProductListingClient.module.css";

const PAGE_SIZE = 24;
const BASE_PATH = "/san-pham/";

export function ProductListingClient({ products }: { products: Product[] }) {
  const searchParams = useSearchParams();
  const raw = searchParams.get("page");
  const currentPage = raw ? parseInt(raw, 10) : 1;
  const totalPages = Math.max(1, Math.ceil(products.length / PAGE_SIZE));

  if (products.length === 0) {
    return <p className={styles.emptyState}>Chưa có sản phẩm nào.</p>;
  }

  if (!Number.isInteger(currentPage) || currentPage < 1 || currentPage > totalPages) {
    return <p className={styles.emptyState}>Trang không tồn tại.</p>;
  }

  const offset = (currentPage - 1) * PAGE_SIZE;
  const slice = products.slice(offset, offset + PAGE_SIZE);

  return (
    <>
      <div className={styles.grid}>
        {slice.map((p) => (
          <ProductCard key={p.slug} product={p} />
        ))}
      </div>
      <Pagination currentPage={currentPage} totalPages={totalPages} basePath={BASE_PATH} />
    </>
  );
}
```

### 4.5 `components/Pagination.tsx` (new)

```tsx
import Link from "next/link";
import styles from "./Pagination.module.css";

interface PaginationProps {
  currentPage: number;
  totalPages: number;
  basePath: string; // e.g. "/san-pham/" or "/danh-muc/chuot-gaming/"
}

export function Pagination({ currentPage, totalPages, basePath }: PaginationProps) {
  if (totalPages <= 1) return null;

  const pageHref = (page: number) =>
    page === 1 ? basePath : `${basePath}?page=${page}`;

  return (
    <nav className={styles.nav} aria-label="Phân trang">
      {currentPage > 1 && (
        <Link className={styles.link} href={pageHref(currentPage - 1)}>
          ← Trước
        </Link>
      )}
      {Array.from({ length: totalPages }, (_, i) => i + 1).map((page) => (
        <Link
          key={page}
          className={`${styles.link} ${page === currentPage ? styles.active : ""}`}
          href={pageHref(page)}
          aria-current={page === currentPage ? "page" : undefined}
        >
          {page}
        </Link>
      ))}
      {currentPage < totalPages && (
        <Link className={styles.link} href={pageHref(currentPage + 1)}>
          Sau →
        </Link>
      )}
    </nav>
  );
}
```

## 8. Risks / open items

- **R1 — Dependency on US00041 + US00042.** If either has not landed, stop at step 2 of §5.
- **R2 — `static/` vs `public/` image path convention.** `Product.images` references `/static/images/products/…` but Next.js serves from `public/`. - **R6 — Pagination ellipsis.** With v1 ≤200 products (~9 pages), rendering all page numbers is fine. Add a `// TODO: add ellipsis for large page counts` comment in `Pagination.tsx` to flag for a future story.
