## 0. Predecessors & dependency posture

US00045 ships the **category surface** of F0004. It depends on three sibling stories in the same feature, all currently un-implemented. The plan below is written so US00045 can ship **after** those siblings land in the order:

1. **US00041 — `formatVnd()` price formatter** (hard dep) — needed by the grid card to render `₫1.200.000`. If `lib/format.ts` does not exist when US00045 is implemented, pause and surface the blocker.
2. **US00042 — `<ProductCard />` component** (hard dep) — the grid is rendered as `<ProductCard />` instances per AC Scenario 4. If `components/ProductCard.tsx` does not exist, pause.
3. **US00043 — Product listing page `/san-pham/`** (soft dep) — US00045 inherits its pagination + canonical strategy verbatim (24/page, `?page=N`, page 1 → bare path, pages 2+ self-canonical). If US00043 lands first, US00045 reuses any extracted pagination helper (`components/Pagination.tsx`, `lib/pagination.ts`). If US00045 lands first, this plan also creates that shared `Pagination` primitive — see §3 D9 below.

Already-shipped predecessors:

- **F0001** — `lib/products.ts` already exposes `getAllProducts()` / `getProductBySlug()` and validates `affiliateUrl` via `assertAffiliateUrl()` at build time. US00045 reads from it; no additions.
- **F0002 — US00024 (`<ShellLayout />`)** — opt-in two-column shell that US00045 wraps the page in.
- **F0002 — US00021 / US00022** — Header + Footer chrome inherited via root `app/layout.tsx`.
- **F0003 — US00031–US00034** — `<AffiliateLink>` + `lib/affiliate.ts` (consumed transitively through `<ProductCard />`).

**Sibling / consumer relationships:**

- **F0009 (SEO & Discoverability)** will harden the SEO surface on top of this story: JSON-LD `BreadcrumbList`, Open Graph tags, sitemap inclusion. US00045 must ship the **canonical strategy** and the **`<h1>` discipline** now so F0009 has the right base.

## 1. Scope

Strictly:

1. Create the dynamic route `app/danh-muc/[category]/page.tsx` — one statically generated page per distinct `category` value present in `content/products/*.json`. Unknown categories → `notFound()`.
2. Create `lib/categories.ts` — single source of truth for category metadata: the Vietnamese display name (used in `<h1>` and the metadata `<title>`) and the 100–200 word Vietnamese intro paragraph per category. Adding a new category requires adding an entry; the build fails loudly if a product JSON references a category absent from this map.
3. Create `app/danh-muc/[category]/not-found.tsx` — Vietnamese 404 surface for unknown categories (per Next.js convention; works alongside `notFound()`).
4. Mount the page inside `<ShellLayout />` (F0002 US00024). Left panel surfaces a list of **other categories** as a navigation aid (one link per sibling category). No client-side filter UI in v1 of this story (filters are US00044's job and live on `/san-pham/`).
5. Render the page with this structure inside `<ShellLayout />`'s `children` slot:
   - One `<h1>` with the Vietnamese category display name.
   - One `<p>` intro paragraph (100–200 words, from `lib/categories.ts`).
   - The filtered grid: products whose `category` exactly equals the route param, rendered via `<ProductCard />` in a responsive 2 / 3 / 4-col layout.
   - Pagination control (`?page=N`) inheriting the US00043 strategy: 24 products per page, page 1 canonical → `/danh-muc/[category]/`, pages 2+ canonical → `/danh-muc/[category]/?page=N`.
   - Empty-state Vietnamese microcopy if the category has zero products.
6. Implement `generateStaticParams()` returning one entry per distinct category in the product catalog.
7. Implement `generateMetadata()` returning the per-page canonical URL, `<title>`, and `description` (the description sources from `lib/categories.ts`). Open Graph + JSON-LD are explicitly deferred to F0009.
8. Add a build-time invariant in `lib/products.ts` (or a new `lib/categories.ts` validator called from `lib/products.ts`) that fails the build if any product's `category` is not registered in `lib/categories.ts`. Mirrors `assertAffiliateUrl()` in spirit (single chokepoint, fail loud, name the offending slug).
9. Create a new responsive grid CSS Module (`app/danh-muc/[category]/category-page.module.css`) for the page-level layout (2/3/4 cols, intro typography). Reuse tokens from `app/globals.css`. `<ProductCard />` owns its own internal styles.

**Explicitly out of scope** (owned by sibling stories or future features):

- The all-products listing `/san-pham/` and its grid — **US00043**.
- The client-side filter / sort UI on `/san-pham/` — **US00044**.
- Brand sub-filtering inside category pages — left for a follow-up if data warrants it; out of v1 of US00045.
- `<ProductCard />` component itself — **US00042**.
- `formatVnd()` helper — **US00041**.
- Product detail page `/san-pham/[slug]/` and related-products row — **US00046 / US00047**.
- JSON-LD `BreadcrumbList`, Open Graph tags, breadcrumb UI — **F0009**.
- `sitemap.xml` entries that list each category route — **F0009**.
- GA4 `affiliate_click` event emission — **F0007** (seam already in `<AffiliateLink>`).
- Real-time price sync, stock indicators, search — v2+.
- Mobile drawer behaviour for the left-panel category nav — already shipped by **US00025** at the shell level; US00045 just places content into the slot.

### 5.1 `lib/categories.ts`

```ts
/**
 * Category registry — F0004 (US00045).
 *
 * Single source of truth for storefront category metadata. Adding a new
 * category means:
 *   (1) adding an entry here, and
 *   (2) tagging product JSON files with the same `category` slug.
 *
 * The product loader (`lib/products.ts`) calls `assertCategoryRegistered()`
 * at build time, so any product JSON whose `category` is not registered
 * here fails `next build` with the offending slug named.
 */

export type CategorySlug = string; // narrow to a union as the catalog grows

export interface CategoryMeta {
  /** URL slug; matches `product.category` in JSON. Kebab-case ASCII. */
  slug: CategorySlug;
  /** Vietnamese display name — used in <h1> and the metadata <title>. */
  name: string;
  /** 100–200 word Vietnamese intro paragraph rendered above the grid. */
  intro: string;
  /** ≤160 char Vietnamese snippet for <meta description>. */
  metaDescription: string;
}

export const CATEGORIES: Record<CategorySlug, CategoryMeta> = {
  "chuot-gaming": {
    slug: "chuot-gaming",
    name: "Chuột gaming",
    intro:
      "Chuột gaming là vũ khí then chốt của mọi game thủ FPS, MOBA hay MMO. " +
      "Tại aff-store, chúng tôi tổng hợp các mẫu chuột chơi game được người " +
      "Việt ưa chuộng nhất hiện nay, từ phân khúc phổ thông dưới 500k cho " +
      "tới các dòng cao cấp với cảm biến quang học chính xác trên 20.000 DPI. " +
      "Mỗi sản phẩm đều có thông tin chi tiết về cảm biến, trọng lượng, số " +
      "nút bấm và độ bền switch để bạn dễ dàng so sánh và chọn mua qua link " +
      "Shopee tiếp thị liên kết.", // ~115 từ — enforce 100–200 manually
    metaDescription:
      "Tổng hợp chuột gaming chính hãng giá tốt — đầy đủ thông số, " +
      "đánh giá nhanh và link mua Shopee tiếp thị liên kết.",
  },
  "tai-nghe": { /* … */ } as unknown as CategoryMeta, // illustrative
  // …more categories added as the catalog grows
};

export function getCategoryMeta(slug: string): CategoryMeta | undefined {
  return CATEGORIES[slug];
}

export function getAllCategorySlugs(): CategorySlug[] {
  return Object.keys(CATEGORIES);
}

const SLUG_REGEX = /^[a-z0-9]+(?:-[a-z0-9]+)*$/;

/**
 * Build-time guard. Called from `lib/products.ts` during JSON validation.
 * Fails the build with a clear error if a product references an unknown
 * category or if the category slug shape is wrong.
 */
export function assertCategoryRegistered(
  category: string,
  productSlug: string,
): void {
  if (!SLUG_REGEX.test(category)) {
    throw new Error(
      `[categories] Product "${productSlug}" has invalid category slug ` +
        `"${category}". Slugs must be lowercase kebab-case ASCII.`,
    );
  }
  if (!(category in CATEGORIES)) {
    throw new Error(
      `[categories] Product "${productSlug}" references unknown category ` +
        `"${category}". Register it in lib/categories.ts first.`,
    );
  }
}
```

### 5.3 `app/danh-muc/[category]/page.tsx` (sketch)

```tsx
import { notFound } from "next/navigation";
import type { Metadata } from "next";
import { getAllProducts } from "@/lib/products";
import { getAllCategorySlugs, getCategoryMeta } from "@/lib/categories";
import { ShellLayout } from "@/components/ShellLayout";
import { ProductCard } from "@/components/ProductCard";
import { CategoryNav } from "@/components/CategoryNav";
import { Pagination } from "@/components/Pagination";
import styles from "./category-page.module.css";

const PAGE_SIZE = 24;

type Params = { category: string };
type Search = { page?: string };

export function generateStaticParams(): Params[] {
  return getAllCategorySlugs().map((category) => ({ category }));
}

function parsePageParam(raw: string | undefined): number {
  if (raw === undefined) return 1;
  const n = Number(raw);
  if (!Number.isInteger(n) || n < 1) return NaN;
  return n;
}

export async function generateMetadata({
  params,
  searchParams,
}: {
  params: Promise<Params>;
  searchParams: Promise<Search>;
}): Promise<Metadata> {
  const { category } = await params;
  const { page: rawPage } = await searchParams;
  const meta = getCategoryMeta(category);
  if (!meta) return {}; // notFound() will run from the page component
  const page = parsePageParam(rawPage);
  const titleBase = `${meta.name} | aff-store`;
  const title = page > 1 ? `${meta.name} – Trang ${page} | aff-store` : titleBase;
  const canonical =
    page > 1
      ? `/danh-muc/${meta.slug}?page=${page}`
      : `/danh-muc/${meta.slug}`;
  return {
    title,
    description: meta.metaDescription,
    alternates: { canonical },
  };
}

export default async function CategoryPage({
  params,
  searchParams,
}: {
  params: Promise<Params>;
  searchParams: Promise<Search>;
}) {
  const { category } = await params;
  const { page: rawPage } = await searchParams;

  const meta = getCategoryMeta(category);
  if (!meta) notFound();

  const page = parsePageParam(rawPage);
  if (!Number.isFinite(page)) notFound();

  const all = getAllProducts();
  const filtered = all
    .filter((p) => p.category === category)
    .sort((a, b) => {
      const dt = new Date(b.publishedAt).getTime() - new Date(a.publishedAt).getTime();
      return dt !== 0 ? dt : a.slug.localeCompare(b.slug);
    });

  const totalPages = Math.max(1, Math.ceil(filtered.length / PAGE_SIZE));
  if (page > totalPages) notFound();

  const slice = filtered.slice((page - 1) * PAGE_SIZE, page * PAGE_SIZE);

  return (
    <ShellLayout leftPanel={<CategoryNav currentSlug={meta.slug} />}>
      <article>
        <header className={styles.header}>
          <h1 className={styles.heading}>{meta.name}</h1>
          <p className={styles.intro}>{meta.intro}</p>
        </header>
        {filtered.length === 0 ? (
          <p className={styles.empty}>Chưa có sản phẩm trong danh mục này.</p>
        ) : (
          <>
            <ul className={styles.grid} role="list">
              {slice.map((product) => (
                <li key={product.slug}>
                  <ProductCard product={product} />
```

### 5.6 `components/Pagination.tsx` (sketch — only if not already shipped by US00043)

```tsx
import Link from "next/link";
import styles from "./Pagination.module.css";

export interface PaginationProps {
  /** Bare path WITHOUT trailing slash or query string, e.g. "/danh-muc/chuot-gaming". */
  basePath: string;
  currentPage: number;
  totalPages: number;
}

export function Pagination({ basePath, currentPage, totalPages }: PaginationProps) {
  if (totalPages < 2) return null;

  const hrefFor = (p: number) => (p === 1 ? `${basePath}/` : `${basePath}/?page=${p}`);

  return (
    <nav aria-label="Phân trang" className={styles.nav}>
      <ul className={styles.list}>
        {currentPage > 1 && (
          <li>
            <Link href={hrefFor(currentPage - 1)} rel="prev">
              Trang trước
            </Link>
          </li>
        )}
        {Array.from({ length: totalPages }, (_, i) => i + 1).map((p) => (
          <li key={p}>
            {p === currentPage ? (
              <span aria-current="page">{p}</span>
            ) : (
              <Link href={hrefFor(p)}>{p}</Link>
            )}
          </li>
        ))}
        {currentPage < totalPages && (
          <li>
            <Link href={hrefFor(currentPage + 1)} rel="next">
              Trang sau
            </Link>
          </li>
        )}
      </ul>
    </nav>
  );
}
```

## 10. Risks / open items

- **Sibling-story ordering.** US00045 depends on US00041 (`formatVnd`) and US00042 (`<ProductCard />`); both must land first or in the same PR train. If the implementer encounters US00045 in isolation, pause and surface the dependency rather than recreating either helper locally. The implementation step §6.1 enforces this.
- **`<Pagination />` duplication risk with US00043.** US00043 also needs paginated `<a href="?page=N">` controls. If US00043 ships first, reuse its `components/Pagination.tsx`. If US00045 ships first, the primitive created here becomes the shared one — and US00043's planner should be told to consume it rather than re-implementing. The Pagination component is intentionally generic (`basePath` prop) for exactly this reason.
- **Category registry content vs. catalog evolution.** Authors must edit `lib/categories.ts` whenever they introduce a new `category` value in product JSON. The build fails loudly if they forget — which is the desired UX — but the failure happens at `next build`, not at JSON edit time. Mitigation: documenting the invariant under `CLAUDE.md` "Module conventions" and on the `lib/categories.ts` file header.
- **Intro paragraph word-count drift.** The 100–200 word constraint is enforced manually at authoring time. A linter rule that counts words inside string literals is out of scope. Mitigation: a `// 100–200 words` comment next to each `intro` field and a manual word-count during code review.
- **Out-of-range `?page=N` returning 404 vs. silently clamping.** Spec § Scenario 5 (US00043) and § Scenario 5 (US00045) say `notFound()`. This plan honors that. Trade-off: bookmarked deep links become 404s after products are removed. Spec § Edge case "Visitor lands on a paginated category URL after the underlying products change" acknowledges this as acceptable v1 behavior.
- **Canonical URL — relative vs. absolute.** This plan ships **relative** canonicals (`/danh-muc/<slug>` / `/danh-muc/<slug>?page=N`). Most SEO tools and Google accept relative canonicals, but F0009 may want to promote them to absolute using `NEXT_PUBLIC_SITE_URL`. Mitigation: F0009 can centralize a `canonicalize(path)` helper and have all `generateMetadata` consumers route through it. US00045 doesn't pre-empt that decision.
- **Empty-category render with `<h1>` + intro present.** Spec says the page renders the empty-state message; this plan also renders the `<h1>` and the intro. Reasoning: the page is still the canonical landing for that category and should be useful to Google even with zero current products (thin content, but content nonetheless). If the reviewer disagrees and wants a minimal "Coming soon" page instead, flip §5.3's empty branch.
- **`<CategoryNav />` content decision.** This plan omits the current category from the nav list (it's a "sibling categories" list, not a tab strip). An alternative is to include all categories with the current one marked `aria-current="page"`. Either works; the chosen design is simpler. Easy to switch in a follow-up.
- **Trailing slash on canonical / `basePath`.** This plan uses `/danh-muc/<slug>` (no trailing slash) for the canonical metadata field and `/danh-muc/<slug>/` (with slash) for hrefs in `<CategoryNav>` and in `Pagination` (`${basePath}/`). Next.js by default normalizes trailing slashes at the boundary, but inconsistent canonical-vs-href can dilute SEO. **Mitigation:** confirm against `next.config.ts` (`trailingSlash` is not set, so default = `false`) and standardize on no-trailing-slash in both places during implementation. Spell this out explicitly in code comments.
- **No tests.** The repo currently has no unit-test framework. Behavior is verified at build time (`next build` exits 0) and via the browser smoke test. If a test framework lands later, the highest-value targets to back-fill are `assertCategoryRegistered()` and `parsePageParam()` in `lib/categories.ts` / page file.
- **Working tree currently shows untracked PNG screenshots and a `.playwright-cli` folder.** Per the gitStatus snapshot. These are unrelated to US00045 and must not be staged. The PR diff should only contain files listed in §4.1 + §4.2.
