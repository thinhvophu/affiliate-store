## 3. Approved Decisions

| # | Decision | Choice | Rationale |
|---|---|---|---|
| D1 | Alias shape: explicit per-folder (`@/components/*`, `@/lib/*`, …) **vs** generic root (`@/*` → `./*`) | **Generic `@/*` → `./*`** | Idiomatic for Next.js (matches `create-next-app`). Subsumes all four required aliases; satisfies the spec because `@/components`, `@/lib`, `@/content`, `@/types` all resolve correctly. Adding a new top-level folder later doesn't require touching tsconfig. |
| D2 | Bundler-side alias config in `next.config.ts` | **Skip — rely on Next.js's automatic tsconfig `paths` integration** | Next.js (15/16) and Turbopack/webpack honor `tsconfig.json#paths` natively when `moduleResolution: bundler` is set. Duplicating in `next.config.ts` creates the exact "alias drift" risk called out in the spec's edge cases. |
| D3 | Empty-folder marker | **`.gitkeep` (zero-byte file) in each empty folder** | Standard convention; nothing else committed for now. |
| D4 | Touchstone for AC Scenarios 2 & 3 | **Temporary import in `app/page.tsx` from a temp `lib/touchstone.ts`**, run `tsc --noEmit` + `next build`, then **revert both files** before commit | Mirrors how US00011 verified TS strict mode (inject → verify → revert). Keeps the committed tree clean — no orphan touchstone code lingering for future stories to step around. |
| D5 | Branch / commit target | **Commit directly to `main`** (matches US00011's pattern in `git log`) | Project doesn't appear to use feature branches yet. Push to `origin/main`. |
| D6 | `static/` vs `public/` reconciliation | **Out of scope for US00012 — just create `static/` per spec.** Note the open question (Section 8). | Next.js only auto-serves `public/`. Resolving how `static/images/...` URLs actually get served belongs to US00015/US00016 (loaders + assets) or F0002 (product detail). |

## 8. Risks / open items

- **`static/` is not auto-served by Next.js.** Spec references `static/images/...` as the asset root, but only `public/` is auto-served. We're creating the folder per spec; how URLs map to it is a deliberate open question for US00015 / F0002. Flag, don't solve here.
- **Path-alias drift** is the spec's named edge case. Mitigated by D2 (single source of truth = `tsconfig.json#paths`; bundler reads it automatically). If any future story adds explicit aliases to `next.config.ts`, that introduces the drift risk and should be challenged.
- **`.gitkeep` lifetime.** Each `.gitkeep` should be deleted by the story that first puts real content into the folder (US00013 → `content/`, US00014 → `types/`, US00015 → `lib/`, etc.). Not a US00012 problem, but worth calling out in the implementing PRs.
